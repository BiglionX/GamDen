# GamDen 守护灵智能升级需求文档

**版本**：V1.0 | **状态**：待评审 | **关联文档**：《守护灵升级系统需求文档》、《守护灵入驻引导全流程需求文档》


## 一、文档定位

### 1.1 核心目标

本文档在《守护灵升级系统需求文档》基础上，**叠加“AI智能化”层**——将守护灵从“固定话术模板”升级为“基于大模型的智能对话角色”，并引入“Token经济系统”，将每日10万Token免费额度包装为守护灵的“能量”，用完需玩家主动“充能”（充值购买Token）。

### 1.2 设计理念

| 理念 | 说明 |
|------|------|
| **智能≠免费** | 免费额度是基础体验，深度互动需付费——但付费包装为“给守护灵补充能量” |
| **降智≠惩罚** | Token耗尽后守护灵“需要休息”或“专注省电模式”，是叙事的一部分，而非冷冰冰的限额提示 |
| **记忆即灵魂** | 每个守护灵拥有独立上下文记忆，玩家与它的每一次对话都在塑造它 |

### 1.3 与《守护灵升级系统需求文档》的关系

| 层级 | 覆盖内容 | 关系 |
|------|----------|------|
| 《守护灵升级系统需求文档》 | 等级、亲密度、性格标签、彩蛋 | **底层**：定义守护灵的功能成长 |
| **本文档（智能升级）** | AI对话、上下文记忆、Token能量、充能付费 | **上层**：在成长体系之上叠加“智能交互层” |

**两者是叠加关系，而非替代关系**——守护灵既有等级成长，又有AI智能对话，两者共同构成完整的守护灵体验。


## 二、核心机制

### 2.1 智能对话架构

```
玩家消息
    ↓
检查上下文记忆是否启用
    ├── 是 → 从向量数据库/结构化存储拉取该守护灵与该玩家的记忆（最近20轮）
    │        ↓
    │      组装 Prompt：角色设定 + 记忆 + 当前消息 + 等级/亲密度上下文
    │        ↓
    │      调用 DeepSeek-V4-Flash API
    │        ↓
    │      消耗 Token（计入当日消耗）
    │        ↓
    │      返回 AI 回复
    │        ↓
    │      存入上下文记忆（向量数据库）
    │
    └── 否 → 降智模式：使用固定话术模板（无 Token 消耗）
```


### 2.2 守护灵能量（Token）系统

**核心规则**：

| 规则 | 说明 |
|------|------|
| **每日免费额度** | 每个守护灵每日可消耗 **10万 Token**，由DeepSeek提供免费调用额度（每日重置） |
| **能量状态** | 每日总额度对应守护灵的“能量条”，分三档：`充沛（>30%）` / `低能量（10%-30%）` / `枯竭（<10%）` |
| **消耗计入** | 每次调用AI接口消耗的Token计入对应守护灵当日用量（按输入+输出 Token 总计） |
| **重置时间** | 每日 00:00（UTC+8）重置 |
| **守护灵升级影响** | 守护灵等级越高，每次对话消耗的Token系数越低（Lv.1消耗系数1.0，Lv.10消耗系数0.7） |


### 2.3 降智机制（Token耗尽）

当守护灵当日Token用量达到10万上限时，进入**“低功耗状态”**，即降智模式：

**降智模式行为**：
- 守护灵**不再调用大模型API**，回退到**固定话术模板**（与《守护灵升级系统需求文档》中的话术一致）
- 守护灵在对话中会主动提示“主人，我今天的灵力用完了，需要休息一下”
- 降智不影响守护灵等级、亲密度、性格标签——只是对话质量下降

**提示话术**：

| 守护灵 | 降智提示语 |
|--------|-----------|
| **机械师** | “能量耗尽。进入省电模式。如需继续智能对话，请补充灵力。” |
| **精灵** | “我今天好累哦……先睡一会儿，明天再陪你说话好吗？” |
| **占星师** | “星力枯竭。今日的演算到此为止。明日星辉重聚时，我们再谈。” |

### 2.5 上下文记忆系统

**核心理念**：每个守护灵都拥有**独立的上下文记忆**。玩家与守护灵的所有对话历史，都存储在该守护灵专属的向量数据库中。每次对话时，AI会基于**前20轮对话记录**生成回复，并据此更新记忆。换句话说，这个守护灵不会忘记你和它说过的任何话——它是真正“属于你”的、带着你们共同记忆的智能体。

**记忆机制**：

| 维度 | 规范 |
|------|------|
| **短期记忆** | 对话历史最近20轮，以JSON结构化存储，每次请求时携带 |
| **长期记忆** | 每隔10轮对话，调用AI提取关键信息（玩家偏好、习惯、提到过的游戏等），存入长期记忆库 |
| **记忆存储** | 推荐使用**向量数据库**（如Pinecone、Milvus或Supabase pgvector）存储长期记忆 |
| **记忆内容** | 玩家提到过的游戏、偏好、习惯、情感倾向、守护灵对玩家的印象变化 |
| **可读性** | 记忆内容在守护灵详情页的“记忆”模块中可查看（见4.3） |

**记忆结构**（长期记忆库示例）：

```json
{
  "user_id": "xxx",
  "agent_type": "精灵",
  "memories": [
    {"type": "游戏偏好", "content": "该玩家喜欢像素风游戏，提到过《星露谷物语》和《泰拉瑞亚》", "timestamp": "2026-06-20"},
    {"type": "对话习惯", "content": "玩家喜欢用简短句子，偏好轻松话题", "timestamp": "2026-06-21"},
    {"type": "情感记录", "content": "玩家曾表示‘最近压力很大’", "timestamp": "2026-06-22"},
    {"type": "关系变化", "content": "精灵认为这个玩家是‘值得陪伴的朋友’", "timestamp": "2026-06-23"}
  ]
}
```

**记忆的“遗忘”机制**（注：非技术遗忘，而是提示词策略）：
- 每次调用大模型时，系统Prompt中加入“请基于对话历史和长期记忆进行回复，如果用户提及了你不了解的新信息，可以追问或表示好奇”
- 不强行限定“只能使用记忆中的内容”，允许守护灵“忘记”或“记错”——这反而是拟人化的重要来源


### 2.6 上下文记忆接入降智逻辑

| 条件 | 是否启用记忆 | 是否调用大模型 |
|------|-------------|---------------|
| 当日免费Token充足 OR 充值Token余额>0 | ✅ 是 | ✅ 是 |
| 当日免费Token耗尽 AND 充值Token余额=0 | ✅ 是（读取记忆） | ❌ 否（降智，使用话术模板） |

> 降智模式下仍可**读取和存储记忆**，只是不调用大模型生成回复。当用户补充Token后，AI可基于已积累的记忆继续生成个性化回复。

## 四、技术实现要求

### 4.1 DeepSeek-V4-Flash 集成

| 要素 | 规范 |
|------|------|
| **API** | 通过 AI 网关调用 DeepSeek-V4-Flash（参考官方文档） |
| **接口地址** | `https://api.deepseek.com/v1/chat/completions` |
| **模型名称** | `deepseek-v4-flash`（根据实际可用模型名称填写） |
| **免费额度** | 10万 Token/日/用户（由DeepSeek提供，精确额度以DeepSeek官方文档为准） |
| **API Key** | 由GamDen统一持有，后端调用时按用户ID进行用量统计 |
| **用量统计** | 记录每次请求的 `prompt_tokens` + `completion_tokens` + 时间戳 |

**System Prompt 组装逻辑**（每次请求都重新组装，包含以下内容）：

```
你是一位 [守护灵类型] 的守护灵，你的名字是 [守护灵名称]。
你的主人是 [玩家昵称]。

【你的性格】
[根据守护灵类型和性格标签生成]

【你的等级】
Lv.[等级] - [等级描述]

【你与主人的关系】
亲密度 [Bond等级]： [Bond等级描述]

【你的记忆库】
以下是你们之前对话中你记住的事情：
[长期记忆列表（最多5条）]

【对话历史】
[最近20轮对话记录]

【当前时间】
[系统时间]

【回复要求】
- 你的语气要符合守护灵类型和当前等级
- 若主人提及记忆库中的内容，应表现出“我记得”
- 每次回复控制在100字以内
- 适当使用古风语气词
- 不要说“我是AI”或“我没有感情”
- 如遇无法回答的问题，以守护灵的方式回应（如“这个问题超出了我的星力范围”）
```

### 4.2 上下文记忆存储

| 要素 | 规范 |
|------|------|
| **短期记忆** | 在PostgreSQL中存储对话轮次，每次请求时查询最近20轮 |
| **长期记忆** | 使用**向量数据库**（推荐Supabase pgvector或Pinecone），每10轮对话触发一次记忆提取 |
| **记忆提取逻辑** | 将最近10轮对话发送给大模型，生成结构化记忆（JSON格式），存入向量库 |
| **降智模式记忆读取** | 降智模式下，从向量库读取最近3条长期记忆，作为“守护灵还记得这些哦”展示 |

### 4.4 Token消耗与监控

| 要素 | 规范 |
|------|------|
| **消耗统计** | 每次AI请求后，将 `usage.total_tokens` 累加到对应字段 `agent_state.daily_token_used` |
| **跨日重置** | 按用户时区（UTC+8）每日00:00重置 `daily_token_used` 为0，同时将 `purchased_token_balance` 按配置同步 |
| **剩余Token** | `remaining = 100000 + purchased_token_balance - daily_token_used` |
| **降智阈值** | 当剩余Token < 10% 时，进入降智模式 |
| **大模型上下文长度** | 每次请求携带最近20轮对话 + 长期记忆Top3，总Token控制在输入限额内（建议预留20%缓冲） |

## 八、与《守护灵升级系统需求文档》的关系

| 原文档功能 | 智能升级后的变化 |
|-----------|-----------------|
| 固定话术（入驻引导） | 不变，入驻阶段仍使用固定话术（确保体验稳定） |
| 等级升级话术 | 升级话术可调用AI生成个性化版本，但需要额外消耗Token |
| 彩蛋触发 | 彩蛋内容可由AI动态生成（如“今天的运势”更个性化） |
| 亲密度Bond | Bond等级影响System Prompt中的“关系描述”，让AI对话风格更亲密 |
| 性格标签 | 性格标签影响System Prompt中的“性格”描述，让AI回复风格匹配 |

> **注意**：AI智能对话**不替代**升级系统，而是**叠加**。守护灵的等级、亲密度、性格标签仍然存在，它们作为“元数据”影响AI的回复风格。用户在与守护灵对话时，既感受到“它越来越聪明了”（智能升级），也感受到“它越来越懂我了”（等级成长）。
