## 1. 产品定位与核心玩法

### 1.1 产品定位（一句话）
GamDen是一个独立于算法的游戏巢穴——玩家通过邀请码入驻获得专属领地，在AI守护灵（Agent）引导下建立游戏社交圈，通过邀请好友扩张领地版图。

**核心价值**：
- 反算法：内容展示按时间线/地理位置，不依赖推荐算法
- 领地感：每个玩家拥有唯一坐标和可视化领地
- 社交链：邀请关系决定领地相邻关系，形成聚居区

### 1.2 核心玩法流程（用户路径）
```
新用户入驻 → 选择守护灵 → 分配随机领地坐标 → 进入地图看到邻居 → 
完成每日任务（签到/发帖）提升等级 → 领地元素图进化 → 
邀请好友入驻（好友领地在旁边） → 邀请满3人解锁个人专属小程序 → 
持续扩张形成聚居区
```

## 2. 功能范围与边界定义

### 2.1 V1.0功能范围（明确边界）

| 模块 | V1.0实现功能 | V2.0规划 | 边界说明 |
|------|--------------|----------|----------|
| **用户系统** | 邀请码注册+手机号/微信登录、个人资料（头像、签名20字、守护灵）、账号安全（修改密码、注销） | 邮箱注册、二次验证 | V1.0不支持邮箱注册；签名必须过AI内容安全 |
| **领地系统** | 注册时随机分配坐标（保证与邀请人在±10格内）、领地升级Lv.1~Lv.5、5套固定元素图（按守护灵类型） | Lv.6~Lv.10、自定义皮肤、装饰商城 | 坐标范围：-1000~+1000；坐标冲突自动向右偏移+1 |
| **地图系统** | 查看坐标±10格范围内邻居、无人区野兽潮自动防御（无需手动操作）、防御成功获得金币（50~200） | 手动攻击、组队讨伐、扩大视野 | V1.0仅展示威胁状态，防御为自动判定 |
| **Agent（守护灵）** | 3种类型（机械师/精灵/占星师）注册时选择（不可更改）、固定话术模板+条件触发（签到/升级/邀请成功）、系统通知拟人化（Agent头像+语气） | 大模型驱动、个性化记忆、更多守护灵类型 | V1.0不接入大模型API，话术直接写死在配置表 |
| **邀请裂变** | 每个用户唯一邀请码、邀请进度追踪、邀请≥3人自动生成个人小程序（二维码+链接） | KOL专属码、邀请排行榜、邀请奖励梯度 | 小程序为引流工具，所有深度交互引导回App |
| **IM通讯** | OpenIM二改基础版（单聊+群聊+自定义消息）、每个俱乐部对应一个OpenIM群组、Agent系统通知（带头像和语气） | Agent自动介入聊天、语音房 | V1.0不修改OpenIM核心源码，仅通过Webhook对接 |
| **俱乐部（贴吧）** | 按游戏名称创建（需Lv.2以上）、发帖（500字）/回帖（200字）、管理员指定、查看活跃度 | 俱乐部排行榜、香火值竞赛 | 所有UGC必须过AI内容安全API |
| **百货（商城）** | 虚拟道具（金币兑换：头像框200金币、chat气泡150金币、特殊签名300金币30天） | 实体周边、盲盒、物流系统 | V1.0无人民币支付，金币与人民币不可兑换 |
| **展会（游戏推荐）** | 仅"即将开市"占位图 | 游戏上架、评测、下载跳转 | V1.0不实现 |
| **小程序** | 个人领地展示页（领地缩略图、邀请进度、签名10字、入驻天数、最新动态）、分享小程序码、所有交互引导回App | 增加更多互动功能 | 小程序是引流工具，不是独立产品 |
| **后台管理** | 用户管理（查看、冻结、邀请关系链）、领地与地图监控（热力图、手动调整坐标、配置野兽潮参数）、内容审核（AI初审+人工复审）、俱乐部管理、数据看板（DAU、留存、K因子等） | 游戏管理、订单管理、分成结算 | 必须接入腾讯云/阿里云内容安全API |

### 2.2 V1.0明确不包含的功能
- ❌ 游戏联运（游戏下载、分成结算）
- ❌ 实体商品销售与物流
- ❌ 语音房/直播
- ❌ 大模型驱动的Agent对话
- ❌ 手动攻击野兽/组队讨伐
- ❌ 领地装饰商城
- ❌ 多语言/国际化

## 3. 用户角色与权限说明

### 3.1 用户角色定义

| 角色名称 | 角色标识 | 定义 | 权限概要 |
|----------|----------|------|----------|
| **游客** | `guest` | 未注册用户，通过营销H5或小程序了解产品 | 仅可查看营销H5和小程序展示页，不可交互 |
| **普通玩家** | `player` | 已注册用户，拥有领地和基础功能权限 | 可使用所有基础功能（领地、地图、邀请、发帖、IM） |
| **高级玩家** | `vip` | Lv.3以上或邀请≥5人的玩家 | 额外拥有创建俱乐部权限（需Lv.2以上） |
| **俱乐部管理员** | `club_admin` | 由运营指定，管理特定俱乐部 | 可删除俱乐部内的帖子、管理成员 |
| **运营人员** | `operator` | 后台管理系统用户，负责内容审核、用户管理 | 可查看所有数据、审核内容、冻结用户 |
| **超级管理员** | `super_admin` | 拥有所有权限，负责系统配置 | 可进行系统配置、查看所有数据、管理运营人员 |

### 3.3 权限校验规则
1. **API层校验**：所有API请求必须在Header中携带JWT Token，服务端校验用户角色和权限
2. **前端展示校验**：根据用户角色隐藏无权限的按钮和入口
3. **数据权限**：普通玩家只能查看自己的数据，运营人员可查看所有数据
4. **操作权限**：删除/冻结等敏感操作必须记录操作日志（operator_id, action, target, timestamp）

## 4. 关键业务流程梳理

### 4.1 用户注册与入驻流程

```
[营销H5] 用户访问 → 填写设备+常用游戏 → 提交申请
                                              ↓
                                    [后台] 运营审核（手动/自动）→ 生成邀请码 → 短信/邮件通知用户
                                              ↓
[App] 用户下载App → 输入邀请码 → 选择守护灵 → 注册成功
                                              ↓
                                    [系统] 分配随机领地坐标（保证与邀请人在±10格内，如邀请人不存在则全图随机）
                                              ↓
                                    [系统] 初始化领地Lv.1、金币（初始100）
                                              ↓
                                    [Agent] 发送欢迎语（按守护灵类型）
```

**关键规则**：
- 邀请码有效期：30天，过期需重新申请
- 坐标分配算法：优先分配在邀请人周围±10格范围内，如无空位则随机分配；如随机坐标已被占用，自动向右偏移+1 until找到空位
- 坐标范围：-1000~+1000（X和Y），保证不重叠

### 4.2 领地升级流程

**经验值获取规则**：

| 行为 | 经验值 | 每日上限 | 备注 |
|------|--------|----------|------|
| 每日签到 | 10 | 10 | 连续签到额外+5/天（上限30） |
| 发帖 | 5 | 50 | 被删除帖子扣除相应经验 |
| 回帖 | 2 | 20 | - |
| 邀请好友入驻 | 50 | 无 | 好友需活跃7天以上才计入（is_active=true） |
| 防御野兽潮 | 20 | 200 | - |

**等级阈值与解锁条件**：

| 等级 | 所需经验 | 解锁条件 | 元素图（机械师/精灵/占星师） |
|------|----------|----------|------------------------------|
| Lv.1 | 0 | 注册即得 | ⚙️齿轮 / 🌱树苗 / 🔮星尘 |
| Lv.2 | 100 | - | 🔧铁砧 / 🌿小树林 / ✨星辉 |
| Lv.3 | 300 | 邀请1人 | ⛏️铁匠铺 / 🏡木屋 / 🔭观星台 |
| Lv.4 | 600 | 邀请3人 | 🔥锻炉 / 🏘️石屋群 / 🌌星门 |
| Lv.5 | 1000 | 活跃14天+邀请5人 | ⚒️铸造厂 / 🏯小型寨落 / 🌀星阵 |

### 4.3 邀请裂变与小程序的衔接流程

```
[用户A] 分享邀请码/链接 → [用户B] 输入邀请码注册成功
                                              ↓
                                    [系统] 记录邀请关系（invite_records表，is_active默认false）
                                              ↓
                                    [用户B] 活跃7天后 → [系统] 更新is_active=true
                                              ↓
                                    [用户A] 查看邀请进度（已邀请X人，仅计数is_active=true的）
                                              ↓
                                    已邀请≥3人？
                                              ↓
                                    [系统] 自动生成个人小程序码和链接（防止重复生成：检查mini_program_generated字段）
                                              ↓
                                    [用户A] 分享小程序码到微信群/朋友圈
                                              ↓
                                    [新用户] 扫码 → 进入用户A的领地展示页
                                              ↓
                                    点击任意按钮？
                                              ↓
                                    [小程序] 弹窗"下载GamDen App，加入巢穴"
                                              ↓
                                    [新用户] 下载App → 输入用户A的邀请码注册
```

**关键规则**：
- 邀请进度条显示"已招募X/10名拓荒者"（目标10人，但解锁小程序仅需3人）
- 小程序页面内容：领地缩略图（当前等级）、邀请进度条、用户自定义签名（10字）、入驻天数、最新动态
- 小程序所有交互按钮统一引导回App，不在小程序内实现深度交互
- 防止刷邀请：同一设备不可重复注册，被邀请人需活跃7天才计入进度

### 4.4 内容发布与审核流程

```
[用户] 发布内容（签名/发帖/回帖）
                                              ↓
                                    [系统] 调用腾讯云内容安全API（AI初审）
                                              ↓
                                    AI审核通过？
                                   ↓                    ↓
                                是（置信度>90%）         否（置信度≤90%或敏感）
                                ↓                       ↓
                    [系统] 内容发布成功    [系统] 内容进入人工复审池（status=pending）
                                ↓                       ↓
                    [用户] 内容可见           [运营] 人工审核（approve/reject）
                                                        ↓
                                              审核通过？          审核拒绝？
                                              ↓                   ↓
                                    [系统] 内容发布成功    [系统] 通知用户+记录违规（第一次警告 → 第二次禁言24小时 → 第三次冻结账号）
```

**审核规则**：
- 用户签名：≤20字符，AI初审（置信度>90%直接通过，≤90%进入人工复审）
- 俱乐部帖子：≤500字符，AI初审（同上）
- 回帖：≤200字符，AI初审（高风险才进入人工复审）
- 敏感词库：政治敏感、色情、暴力、人身攻击、商业广告
- 违规处理梯度：第一次警告 → 第二次禁言24小时 → 第三次冻结账号

### 4.5 野兽潮防御流程

```
[系统] 定时任务（每小时检查一次）
                                              ↓
                                    检查无人区（无玩家区域）？
                                              ↓
                                    随机触发野兽潮（概率20%，等级1~5对应威胁度）
                                              ↓
                                    [系统] 生成野兽潮事件（beast_events表）
                                              ↓
                                    附近有玩家领地（±20格）？
                                              ↓
                                    [系统] 推送威胁通知给附近玩家（Agent拟人化话术）
                                              ↓
                                    [用户] 查看野兽潮状态（/api/map/beast-status）
                                              ↓
                                    自动防御（无需手动操作，基于领地等级判定）
                                              ↓
                                    防御成功？（Lv.3以上100%成功，Lv.3以下70%成功）
                                    ↓                    ↓
                                是                      否
                                ↓                       ↓
                    [系统] 发放金币奖励（50~200，基于野兽潮等级）    [系统] 领地等级-1（最低Lv.1）
                    [Agent] 发送防御成功通知                      [Agent] 发送防御失败通知
```

**关键规则**：
- 野兽潮刷新参数：时间（每小时）、频率（20%概率）、强度（1~5级），由运营在后台配置（beast_config表）
- 防御失败处理：Lv.3以下领地有30%概率防御失败，失败后领地等级-1（最低Lv.1）
- V2.0才支持手动攻击和组队讨伐

### 4.6 金币系统流程

**金币获取途径**：

| 途径 | 数量 | 限制 | 备注 |
|------|------|------|------|
| 注册奖励 | 100 | 一次性 | - |
| 每日签到 | 10 | 每日1次 | 连续签到额外+5/天（上限30） |
| 发帖 | 5 | 每日上限50 | 被删除帖子扣除相应金币 |
| 回帖 | 2 | 每日上限20 | - |
| 邀请好友入驻 | 50 | 无 | 好友需活跃7天以上（is_active=true） |
| 防御野兽潮 | 50~200 | 无 | 基于野兽潮等级（Lv.1=50，Lv.5=200） |
| 每周活跃奖励 | 100 | 每周1次 | 连续7天登录 |

**金币消耗途径**：

| 途径 | 消耗 | 有效期 | 备注 |
|------|------|--------|------|
| 兑换头像框 | 200 | 永久 | - |
| 兑换chat气泡 | 150 | 永久 | - |
| 兑换特殊签名（30字） | 300 | 30天 | 需Lv.3以上 |
| V2.0规划 | - | - | 实体周边、盲盒 |

**关键规则**：
- 金币与人民币不可兑换（V1.0）
- 金币余额上限：99999
- 金币不可转账/赠送
- 金币余额不足时，兑换操作返回错误码1003

## 5. 非功能性需求

### 5.1 性能需求

| 指标 | 要求 | 备注 |
|------|------|------|
| **App启动时间** | ≤2秒（冷启动） | 包括初始化OpenIM |
| **页面加载时间** | ≤1秒 | 地图页≤2秒（需加载邻居数据） |
| **API响应时间** | ≤500ms（P95） | 复杂查询≤1秒 |
| **并发用户数** | 支持1000在线用户 | V1.0目标DAU 500 |
| **数据库连接池** | 最大100连接 | MySQL连接池配置 |
| **Redis缓存命中率** | ≥90% | 用户会话、领地信息缓存 |
| **图片加载** | WebP格式，≤200KB | 领地元素图、头像 |
| **H5打开速度** | ≤1秒 | 营销H5首屏渲染 |

### 5.2 安全性需求

| 类别 | 需求 | 实现方案 |
|------|------|----------|
| **身份认证** | JWT Token，有效期7天，支持刷新 | Token存储在SecureStore（App）、HttpOnly Cookie（Web） |
| **密码存储** | bcrypt加密，salt rounds≥10 | 不可明文存储 |
| **API防护** | 防CSRF、XSS、SQL注入 | ORM参数化查询、输入验证 |
| **敏感数据** | 手机号、邀请码脱敏展示 | 手机号显示138****1234 |
| **内容安全** | 所有UGC必须经过AI内容安全API | 腾讯云/阿里云内容安全API |
| **邀请码防刷** | 邀请码一次性使用，绑定设备ID | 同一设备不可重复注册 |
| **限流** | API限流：100次/分钟/用户 | Redis计数 |
| **日志审计** | 所有敏感操作记录日志 | 用户冻结、内容删除、系统配置变更 |
| **数据备份** | 每日自动备份数据库 | 保留7天备份 |
| **HTTPS** | 全站HTTPS | SSL证书配置 |

### 5.3 可用性需求

| 指标 | 要求 | 备注 |
|------|------|------|
| **系统可用性** | ≥99%（V1.0） | 允许计划内维护（凌晨2~4点） |
| **数据持久性** | ≥99.9% | 数据库主从备份 |

## 6. Agent（守护灵）V1.0方案

### 6.1 守护灵类型与话术模板

V1.0暂不接入大模型API，采用"固定话术模板 + 条件触发"，降低开发成本和API费用。

| 守护灵类型 | 欢迎语 | 签到提醒 | 邀请成功 | 等级提升 |
|------------|--------|----------|----------|----------|
| **机械师** | "检测到新信号。欢迎归巢，编号[ID]。" | "今日补给未领取，建议执行。" | "新坐标已校准，盟友已就位。" | "系统升级完成。你现在更强了。" |
| **精灵** | "呜~森林在颤动。你终于来了。" | "太阳晒到树梢了，该浇水啦~" | "新芽破土了！你的盟友就在旁边。" | "古树长高了。你正在变得强大。" |
| **占星师** | "星辰轨迹中，我看到了你的到来。" | "星盘显示，今日宜签到。" | "命运之线已连接，盟友就在你身旁。" | "星力涌动。你的境界突破了。" |

### 6.2 触发条件与实现方式

| 触发事件 | 触发时机 | 实现方式 | 备注 |
|----------|----------|----------|------|
| 欢迎语 | 用户注册成功 | 服务端触发，写入agent_dialogues表，通过OpenIM自定义消息推送 | - |
| 签到提醒 | 每日9:00（可配置） | 定时任务，检查昨日未签到用户，推送Agent话术 | 仅推送一次 |
| 邀请成功 | 被邀请人注册成功 | 服务端触发，推送给自己 | - |
| 等级提升 | 经验值达到阈值 | 服务端触发，推送给自己 | - |
| 防御成功/失败 | 野兽潮防御结算 | 服务端触发，推送给自己 | - |

### 6.3 技术实现要点
- 话术模板直接写死在客户端或后台配置表（agent_templates表）
- 服务端根据触发事件和守护灵类型，从配置表读取对应话术
- 通过OpenIM自定义消息发送（带守护灵头像和语气）
- 所有Agent话术必须记录到agent_dialogues表，用于V2.0大模型训练数据分析

## 7. 多端策略总结

| 终端 | V1.0实现范围 | 核心目的 | 技术栈 |
|------|--------------|----------|--------|
