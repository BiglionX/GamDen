# GamDen 守护灵入驻引导全流程需求文档

**版本**：V1.0 | **状态**：待评审 | **关联文档**：《PRD-V1.0-核心玩法与世界观》、《UI/UX设计规范-守护灵系统》、《游客模式体验设计需求文档》


## 一、文档定位

### 1.1 核心目标

本文档定义了守护灵（Agent）在**新用户从“首次打开App”到“完成入驻并开始使用”全过程中**的角色定位、交互时机、话术规范和视觉呈现标准。

**核心理念**：守护灵不是入驻流程的“客服”或“引导工具”，而是用户对GamDen产生情感连接的第一推动力。入驻流程的每一处UI交互，都被“守护灵叙事”重新包裹，让用户感觉自己在与一个角色共同经历一段旅程，而非在操作一个App。

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **叙事先行** | 每一步操作都有守护灵的台词陪伴，功能界面退居其次 |
| **性格统一** | 三类守护灵（机械师/精灵/占星师）各有一套完整的叙事语言体系 |
| **低门槛高共鸣** | 用户不需要做任何额外操作，只需要“被陪伴” |
| **渐进式亲近** | 随着入驻阶段推进，守护灵的语气从陌生→亲切→归属感逐步递进 |

### 1.3 与其他文档的关系

| 文档 | 与本文档的关系 |
|------|---------------|
| 《PRD-V1.0》 | 本文档是PRD中“守护灵系统”章节的**完整展开** |
| 《UI/UX设计规范》 | 本文档中的弹窗、气泡样式引用该规范 |
| 《游客模式体验设计需求》 | 本文档中的“软引导”部分承接游客模式的设计 |

> **关键说明**：本文档独立于《游客模式体验设计需求》，后者定义了“游客能否做某事”的权限边界，本文档则定义了“守护灵在入驻过程中说了什么、什么时候说”。

### 2.2 各阶段守护灵核心使命

| 阶段 | 守护灵核心使命 | 情感目标 |
|------|---------------|---------|
| **阶段一：游客浏览** | 打破沉默，建立初步连接 | 让用户产生“这里有个活的东西在等我”的感觉 |
| **阶段二：注册** | 完成自我介绍，确认绑定关系 | 让用户感觉“我在和它对话，不是在填表” |
| **阶段三：守护灵选择** | 深化关系，确立契约 | 让用户产生“我选择了它，它以后就是我的了”的归属感 |
| **阶段四：领地落地** | 欢迎归巢，开启陪伴 | 让用户产生“这个世界是活的，它是为我准备的”感知 |


## 三、详细交互设计

### 3.1 阶段一：游客浏览 → 建立连接

**触发条件**：用户首次打开App，以游客身份浏览地图 ≥ 30秒。

**交互设计**：

| 要素 | 规范 |
|------|------|
| **表现形式** | 页面底部从下往上缓慢浮起的圆形头像气泡（带守护灵头像），持续3秒后自动展开为一句话 |
| **浮起动画** | 缓动函数 `ease-out`，持续800ms |
| **展开后停留时间** | 15秒后自动淡出消失 |
| **点击行为** | 点击气泡 → 展开为完整引导弹窗（见下文） |
| **频次控制** | 同一设备30分钟内最多触发2次 |

**话术库（三选一随机，避免重复感）** ：

| 守护灵 | 软引导气泡文案 |
|--------|---------------|
| **机械师** | “检测到正在执行‘观望’指令。已持续45秒，建议进入‘决策’阶段。” |
| **精灵** | “你站在迷雾边缘很久了。风里有一种熟悉的气息——这里，也曾经是我的家。” |
| **占星师** | “星盘上有一颗新的光点正在靠近。是你，对吗？” |

**完整引导弹窗**（点击气泡后） ：

```
┌─────────────────────────────────────────────────┐
│                                                  │
│           [守护灵头像 - 居中]                    │
│                                                  │
│    “外面的世界那么吵，不如留下个名字，            │
│     我给你一块地。”                              │
│                                                  │
│            [  领取我的领地  ]                     │
│            [  再看看  ]                         │
│                                                  │
└─────────────────────────────────────────────────┘
```

- **弹窗样式**：全屏半透明蒙层 + 居中卡片，卡片背景为“宣纸白”，带古风边框纹理
- **主按钮**：【领取我的领地】——跳转至注册页
- **次按钮**：【再看看】——关闭弹窗，回到地图

### 3.2 阶段二：注册 → 初次相遇

**触发条件**：用户点击【领取我的领地】按钮，进入注册页。

**交互设计**：

| 要素 | 规范 |
|------|------|
| **表现形式** | 注册页背景替换为守护灵对话界面。页面中央为守护灵头像（较大尺寸）+ 对话气泡，表单元素以“半透明浮层”形式置于对话下方 |
| **过渡动画** | 从引导弹窗→注册页的切换，伴随守护灵头像“放大→居中”的平滑动画 |

**话术库**：

| 守护灵 | 初次相遇台词 | 询问名字 |
|--------|-------------|---------|
| **机械师** | “信号同步中。我的编号是[M-07]，GamDen巢穴的守夜人。” | “你的领地坐标已锁定，等待指挥官确认身份——请告诉我你的名字。” |
| **精灵** | “终于，你走过来了。我叫[灵]，这片森林的守护者。” | “你的领地在等你。但在这之前，告诉我你的名字。” |
| **占星师** | “星辰已经等到了它的观星者。我等你很久了。” | “请告诉我，我该用什么名字呼唤你？” |

**注册表单**（半透明浮层，位于对话下方）：

| 元素 | 规范 |
|------|------|
| 输入项1 | 手机号（中国大陆 +86） |
| 输入项2 | 验证码（6位数字，60秒倒计时） |
| 协议 | 《用户协议》《隐私政策》复选框（默认不勾选） |
| 主按钮 | 【归巢】——品牌主色（领地金 #C9A87C），点击后调用注册API |

### 3.3 阶段三：守护灵选择 → 结盟仪式

**触发条件**：用户完成注册，验证通过后自动进入守护灵选择页。

**交互设计**：

| 要素 | 规范 |
|------|------|
| **页面标题** | “你愿意与谁同行？” |
| **副标题** | “它将成为你巢穴的守护者，陪你走过GamDen的每一天。” |
| **选择方式** | 三张卡片（机械师/精灵/占星师），点击后高亮 + 轻触震动反馈 |
| **确认按钮** | 用户选择后，底部出现【结盟】按钮（此前为灰色不可点） |

**卡片内容**：

| 守护灵 | 图标 | 性格标签 | 选择台词（悬停/点击时出现） |
|--------|------|---------|--------------------------|
| **机械师** | ⚙️ | 理性 / 可靠 / 冷静 | “机械不会说谎。我会用数据为你守护每一寸领地。” |
| **精灵** | 🌿 | 温暖 / 感性 / 陪伴 | “风会带来远方的消息。我陪你等每一个邻居到来。” |
| **占星师** | ✨ | 深邃 / 智慧 / 指引 | “星辰的轨迹里，我已经看到了你的未来。跟我来。” |

**选择完成 → 确认关系**（弹窗或全屏过渡）：

| 守护灵 | 结盟确认台词 |
|--------|-------------|
| **机械师** | “指令确认。人机绑定完成。欢迎归队，指挥官。” |
| **精灵** | “你选了我。真好。那么——欢迎回家。” |
| **占星师** | “命运之线，从此缠绕。我陪你看未来的每一颗星。” |

### 3.4 阶段四：领地落地 → 归属感

**触发条件**：用户完成守护灵选择，系统自动分配领地坐标。

**交互设计**：

| 要素 | 规范 |
|------|------|
| **表现形式** | 全屏地图视图，一枚光点（“种子”）从屏幕中央落下，落地后扩散出一圈涟漪，涟漪散去后显示领地图标 |
| **动画时长** | 约1.5秒 |
| **伴随效果** | 轻微震动反馈 + 系统Toast提示：“欢迎归巢，你获得了第XXXXX号领地” |

**守护灵祝贺消息**（领地落地后立即弹出，以OpenIM自定义消息形式呈现）：

| 守护灵 | 领地落地祝贺台词 |
|--------|-----------------|
| **机械师** | “领地已落地，坐标已锁定。这是你的起点，请开始建设。” |
| **精灵** | “你看，这里就是你的领地了。现在它很小，但风会帮你带来邻居。我保证。” |
| **占星师** | “第一个坐标，是命运给你的礼物。以后这里会开满属于你的故事。” |

### 3.5 阶段五：新手任务 → 陪伴成长

**触发条件**：用户首次进入“我的”页面（个人中心）。

**交互设计**：

| 要素 | 规范 |
|------|------|
| **表现形式** | 守护灵以系统消息形式弹出（OpenIM自定义消息），位于页面顶部 |
| **消息样式** | 守护灵头像（小尺寸）+ 对话气泡，气泡带古风边框，背景色为宣纸白 |
| **持续时间** | 常驻显示，用户点击“知道了”或“去看看”后消失 |
| **任务跟踪** | 完成后守护灵再次弹出祝贺 |

**话术库**：

| 守护灵 | 新手任务引导 |
|--------|-------------|
| **机械师** | “建议执行初始任务：浏览集市。寻找领地升级所需资源。是否立即执行？” |
| **精灵** | “要不要先去集市看看？那里有可以让领地变漂亮的东西。我陪你。” |
| **占星师** | “你的第一步，应该是去集市看看那些为你准备的‘星尘’。走吧，我引路。” |

**任务完成后的反馈**：

| 守护灵 | 任务完成祝贺 |
|--------|-------------|
| **机械师** | “任务完成。初步建设已启动。继续前进。” |
| **精灵** | “你去了集市！真好。领地会一天天变好看的。” |
| **占星师** | “第一步已经迈出。星途漫漫，我陪你走。” |


## 四、视觉与交互规范

### 4.1 守护灵头像与视觉元素

| 元素 | 规范 |
|------|------|
| **头像尺寸** | 软引导气泡：40x40pt；对话弹窗：80x80pt；注册页主视觉：120x120pt |
| **头像风格** | 像素风格 + 古风元素，三类守护灵各有专属色系 |
| **对话气泡** | 圆角矩形，带小三角指向守护灵头像，背景色为宣纸白（#F5F0E6），边框为领地金（#C9A87C） |

### 4.2 各形态下的显示规格

| 形态 | 头像尺寸 | 气泡位置 | 额外元素 |
|------|----------|----------|----------|
| **软引导气泡** | 40pt | 底部居中浮起 | 无背景，纯头像+文字 |
| **引导弹窗** | 80pt | 卡片顶部居中 | 全屏蒙层+古风边框卡片 |
| **注册页主视觉** | 120pt | 页面中上区域 | 对话气泡，表单半透明浮层在下 |
| **选择页卡片** | 60pt | 卡片内左侧 | 性格标签+选择台词 |
| **系统通知消息** | 30pt | 页面顶部 | OpenIM自定义消息样式 |

### 4.3 动画与过渡规范

| 场景 | 动画 | 时长 | 缓动函数 |
|------|------|------|----------|
| 软引导气泡浮起 | 从底部上滑+淡入 | 800ms | ease-out |
| 注册页进入 | 守护灵头像放大+居中 | 600ms | ease-in-out |
| 领地落地 | 光点落下+涟漪扩散 | 1.5s | 固定时间轴 |
| 弹窗出现 | 缩放+淡入（0.8→1.0） | 300ms | ease-out |


## 五、技术实现要求

### 5.1 前端（UniApp）

| 要求 | 说明 |
|------|------|
| **状态管理** | 在Pinia store中维护当前守护灵类型（`guardian_type`），所有话术据此渲染 |
| **话术存储** | V1.0话术写死在代码中（作为fallback），同时支持从后端API获取（用于后续运营调整） |
| **触发逻辑** | 软引导气泡的触发计时器放在App.vue的`onLaunch`和`onShow`中 |
| **条件编译** | 小程序端同样保留守护灵引导流程（除“注册”步骤外，小程序无注册功能） |

### 5.2 后端API扩展

| 接口 | 说明 |
|------|------|
| `GET /api/agent/dialogues?scene=onboarding` | 获取入驻阶段话术配置（供前端拉取，V1.0可选） |
| `POST /api/agent/onboarding/complete` | 记录用户完成入驻全流程的时间戳，用于分析流失节点 |

### 5.3 数据埋点

| 事件名称 | 触发条件 | 附加字段 |
|---------|---------|----------|
| `onboarding_guide_bubble_shown` | 软引导气泡弹出 | `guardian_type`, `session_duration` |
| `onboarding_guide_clicked` | 用户点击软引导气泡 | `guardian_type` |
| `onboarding_register_started` | 用户进入注册页 | `guardian_type`（未选择，为null） |
| `onboarding_guardian_selected` | 用户完成守护灵选择 | `guardian_type` |
| `onboarding_territory_landed` | 领地分配动画完成 | `guardian_type`, `territory_coord` |
| `onboarding_completed` | 用户完成新手任务 | `guardian_type`, `total_duration` |
