## 1. 营销H5网页（精简版）

### 1.1 页面流程

```
[用户] 访问营销H5（通过朋友圈分享、公众号推文等渠道）
    ↓
[首屏] 动态古风地图剪影 + 主文案"在算法之外，建一座游戏巢穴" + CTA按钮【领取通关文牒】
    ↓
[用户] 点击【领取通关文牒】
    ↓
[申请表单页] 填写设备型号 + 常用游戏 → 提交
    ↓
[系统] 提交成功，显示"等待审批"提示
    ↓
[后台] 运营审核（手动/自动）→ 生成邀请码 → 短信/邮件通知用户
    ↓
[用户] 收到邀请码 → 点击"已有邀请码？立即下载"
    ↓
[输入邀请码页] 输入邀请码 → 验证通过 → 跳转App下载页
    ↓
[用户] 下载App → 输入邀请码注册 → 入驻成功
```

### 1.2 首屏设计

**视觉元素**：
- 背景：动态古风地图剪影（CSS动画，地图缓缓展开）
- 主文案："在算法之外，建一座游戏巢穴"（字体：宋体，颜色：#D4AF37）
- CTA按钮：【领取通关文牒】（颜色：#D4AF37，悬停效果：放大5%）
- 适配：移动端优先，打开速度 < 1秒

**关键技术**：
- 图片优化：WebP格式，≤200KB
- 懒加载：首屏仅加载关键资源
- CDN加速：静态资源部署到CDN

### 1.3 申请表单页

**表单字段**：

| 字段名 | 类型 | 是否必填 | 验证规则 | 说明 |
|--------|------|----------|----------|------|
| 昵称 | 文本输入框 | 必填 | 2~20字符 | - |
| 手机号 | 手机号输入框 | 必填 | 大陆手机号格式 | 用于接收邀请码 |
| 设备型号 | 下拉选择 | 必填 | 预设选项 | iPhone 13/14/15, Samsung S23/S24, 其他 |
| 常用游戏 | 文本输入框 | 必填 | 2~50字符 | 如：原神、王者荣耀 |
| 邀请码（可选） | 文本输入框 | 选填 | 6位字母数字 | 如已有邀请码，可直接输入 |

**提交逻辑**：
1. 用户填写表单，点击【提交申请】
2. 前端验证：检查必填字段是否填写、格式是否正确
3. 后端验证：检查手机号是否已被注册、邀请码是否有效（如填写）
4. 提交成功：显示"申请已提交，我们将在3个工作日内审核并发送邀请码至您的手机"
5. 提交失败：显示错误提示（如"手机号已被注册"）

### 1.4 审核流程

**自动审核**（推荐）：
- 规则：手机号合法 + 设备型号已选 + 常用游戏非空 → 自动通过
- 通过后：系统自动生成邀请码，发送短信/邮件通知用户
- 通知内容："恭喜！您已获得GamDen内测资格，邀请码：A3B7C9。点击链接下载App：https://gamden.com/download"

**手动审核**（备用）：
- 运营人员在后台查看申请列表
- 审核操作：通过/拒绝
- 通过后：生成邀请码，通知用户
- 拒绝后：通知用户（如"抱歉，您的申请未通过审核"）

### 1.5 已有邀请码入口

**页面设计**：
- 入口：首屏底部小字"已有邀请码？立即下载"
- 点击后：跳转至输入邀请码页面
- 输入框：6位字母数字，自动转为大写
- 验证通过：跳转App下载页
- 验证失败：显示"邀请码无效或已使用"

**App下载页**：
- iOS：跳转App Store（预留，V1.0可能仅为TestFlight）
- Android：下载APK（托管在CDN）
- 微信小程序：跳转小程序（如已发布）

## 2. 微信小程序（个人领地展示版）

### 2.1 触发条件

**自动生成规则**：
- 用户邀请好友 ≥ 3人（且is_active=true）
- 系统自动生成个人小程序码和链接
- 防止重复生成：检查`mini_programs`表的`mini_program_generated`字段

**生成流程**：
1. 系统检测到用户邀请人数 ≥ 3人
2. 调用微信小程序API生成小程序码（`wxacode.getUnlimited`）
3. 将小程序码上传到COS，获取URL
4. 在`mini_programs`表插入记录
5. 通知用户"您的个人小程序已生成，点击查看"

### 2.2 页面内容

**页面路径**：`pages/territory/index?user_id=123456`

**页面元素**：

| 元素 | 说明 | 数据来源 |
|------|------|----------|
| 领地缩略图 | 当前等级的像素图（按守护灵类型） | `users`表 + `territory_evolution`表 |
| 领地名称 | 用户自定义，限10字 | `users`表（需新增`territory_name`字段） |
| 入驻天数 | 注册至今的天数 | `users`表`created_at`字段 |
| 等级 | 领地等级（Lv.1~Lv.5） | `users`表`level`字段 |
| 邀请进度条 | "已招募 X/10 名拓荒者" | `invite_records`表（count is_active=true） |
| 最新动态 | "邻居B刚刚入驻，你的领地又扩大了" | 定时任务生成动态，存储在`territory_dynamics`表 |
| 用户签名 | 个性签名（20字） | `users`表`signature`字段 |
| 分享按钮 | 生成带参数的小程序码，可发到微信群/朋友圈 | 微信小程序API |

**页面设计**：
- 风格：像素风/古风，与App保持一致
- 颜色：主色调#D4AF37（金色），背景#1A1A1A（深色）
- 适配：全面屏适配（iPhone刘海屏、Android挖孔屏）

### 2.3 分享能力

**分享流程**：
1. 用户点击【分享】按钮
2. 系统调用微信小程序API生成带参数的小程序码
3. 参数：`user_id=123456`（标识是哪个用户的领地）
4. 用户将小程序码保存到相册，发到微信群/朋友圈
5. 新用户扫码 → 进入该用户的领地展示页

**小程序码设计**：
- 中心：领地缩略图（当前等级的像素图）
- 周围：装饰性边框（像素风）
- 底部：文案"快来加入我的游戏巢穴"

### 2.4 引流策略

**所有交互按钮统一引导**：
- 按钮文案："打开GamDen App体验完整世界"
- 点击后：显示弹窗"下载GamDen App，加入巢穴"
- 弹窗操作：
  - 【立即下载】：跳转App下载页
  - 【取消】：关闭弹窗

**关键原则**：
- 小程序是引流工具，不是独立产品
- 所有付费和深度交互都回流到App
- 小程序内不可注册、不可发帖、不可IM聊天

### 2.5 数据统计

**关键指标**：

| 指标 | 说明 | 统计方式 |
|------|------|----------|
| 小程序访问量 | 每日小程序访问次数 | 微信小程序数据分析 |
| 分享次数 | 小程序码被分享的次数 | 调用分享API时记录 |
| App下载转化率 | 从小程序下载App的比例 | 下载页加埋点，追踪来源 |
| 注册转化率 | 从小程序下载App后注册的比例 | 注册页加埋点，追踪来源 |

**数据看板**：
- 运营人员在后台查看数据看板（同后台管理系统需求文档6.1）
- 展示：小程序访问量趋势、分享次数趋势、转化率

## 3. 邀请裂变完整链路

### 3.1 链路流程图

```
[用户A] 分享邀请码/链接 → [用户B] 输入邀请码注册成功
    ↓
[系统] 记录邀请关系（invite_records表，is_active=false）
    ↓
[用户B] 活跃7天后 → [系统] 更新is_active=true
    ↓
[用户A] 查看邀请进度（已邀请X人，仅计数is_active=true）
    ↓
邀请≥3人？
    ↓
[系统] 自动生成个人小程序码和链接
    ↓
[用户A] 分享小程序码到微信群/朋友圈
    ↓
[新用户C] 扫码 → 进入用户A的领地展示页
    ↓
[用户C] 点击"打开GamDen App体验完整世界"
    ↓
[用户C] 下载App → 输入用户A的邀请码注册
    ↓
[系统] 记录邀请关系（用户A → 用户C）
    ↓
循环...
```

### 3.2 关键节点说明

**节点1：邀请码分发**
- 用户A的邀请码：6位字母数字（如A3B7C9）
- 分发渠道：微信好友、微信群、朋友圈、QQ、微博
- 分发方式：复制邀请码、分享邀请链接（H5页面，输入邀请码即可注册）

**节点2：邀请进度追踪**
- 用户A查看邀请进度：`/api/invite/progress`接口
- 进度计算：count(invite_records where inviter_id=A and is_active=true)
- 进度展示："已招募 X/10 名拓荒者"（目标10人，但解锁小程序仅需3人）

**节点3：小程序生成**
- 触发条件：is_active=true的邀请人数 ≥ 3
- 生成内容：小程序码（图片URL）、小程序页面路径
- 存储位置：`mini_programs`表

**节点4：小程序分享**
- 用户A点击【分享】按钮
- 系统调用微信小程序API生成带参数的小程序码
- 参数：user_id=A
- 用户A将小程序码发到微信群/朋友圈

**节点5：引流回流**
- 新用户C扫码 → 进入用户A的领地展示页
- 点击任意按钮 → 弹窗"下载GamDen App，加入巢穴"
- 下载App → 输入用户A的邀请码注册

### 3.3 防刷机制

**问题**：用户可能刷邀请码（注册多个账号，骗取小程序生成）

**解决方案**：
1. **设备ID绑定**：同一设备不可重复注册（记录设备ID，注册时验证）
2. **邀请人活跃度要求**：被邀请人需活跃7天才计入进度（is_active=true）
3. **小程序生成限制**：每个用户只能生成一个小程序（mini_program_generated字段）
4. **异常检测**：同一IP短时间内多次注册 → 标记风险，需人工审核

## 4. 渠道推广策略

### 4.1 线上渠道

| 渠道 | 推广方式 | 目标用户 | 成本 |
|------|----------|----------|------|
| 微信公众号 | 发布推文（领地截图+文案） | 公众号粉丝 | 低（自有渠道） |
| 朋友圈广告 | 投放H5链接（"领取通关文牒"） | 游戏玩家 | 中（按点击付费） |
| 游戏论坛（如NGA、贴吧） | 发帖推广（软文） | 核心玩家 | 低（需人工维护） |
| 抖音/快手 | 短视频推广（领地展示） | 年轻玩家 | 高（按曝光付费） |
| KOL合作 | 游戏UP主推广 | UP主粉丝 | 高（按合作费用） |

### 4.2 线下渠道

| 渠道 | 推广方式 | 目标用户 | 成本 |
|------|----------|----------|------|
| 游戏展会 | 展台推广（扫码下载） | 展会观众 | 高（展台费用） |
| 网吧 | 电脑桌面广告（H5链接） | 网吧玩家 | 中（按网吧数量付费） |
| 高校 | 社团合作（扫码下载） | 大学生玩家 | 低（社团赞助） |

### 4.3 裂变 incentives（V2.0规划）

**V1.0**：无物质奖励，仅精神激励（领地升级、解锁小程序）

**V2.0规划**：
- 邀请1人：获得100金币
- 邀请3人：解锁个人小程序
- 邀请5人：获得特殊头像框
- 邀请10人：获得实体周边（如贴纸、徽章）

## 5. 数据分析与优化

### 5.1 关键指标

| 指标 | 说明 | 目标值（V1.0） |
|------|------|----------------|
| H5访问量 | 每日H5访问次数 | 500 |
| H5转化率 | 从H5申请到注册的比例 | 20% |
| 邀请码分发量 | 每日邀请码被分享的次数 | 100 |
| 邀请转化率 | 从邀请码到注册的比例 | 30% |
| 小程序访问量 | 每日小程序访问次数 | 200 |
| App下载转化率 | 从小程序下载App的比例 | 15% |
| 注册转化率 | 从下载App到注册的比例 | 50% |
| K因子 | 平均每个用户邀请多少人 | 0.5（V1.0目标） |

### 5.2 数据分析工具

| 工具 | 用途 |
|------|------|
| 微信小程序数据分析 | 小程序访问量、分享次数、用户画像 |
| Google Analytics（GA） | H5访问量、转化率、用户行为 |
| 腾讯云CLS（日志服务） | API访问日志、错误日志 |
| 自建数据看板 | 整合所有数据，展示关键指标 |

### 5.3 优化策略

**A/B测试**：
- H5首屏文案：测试不同文案的转化率
- CTA按钮颜色：测试不同颜色的点击率
- 邀请进度展示：测试"X/3" vs "X/10"的邀请动力

**迭代优化**：
- 每周查看数据看板，分析关键指标
- 每月召开运营会议，讨论优化策略
- 根据数据调整推广渠道和预算
