# GamDen 俱乐部升级系统需求文档

**版本**：V1.0 | **状态**：待评审 | **关联文档**：《PRD-V1.0-核心玩法与世界观》、《OpenIM集成方案》


## 一、文档定位

### 1.1 核心目标

本文档定义了GamDen俱乐部系统从“静态群组”升级为“覆盖玩家全维度兴趣的可生长活跃社区”的完整机制，解决“俱乐部太少，无法覆盖足够多玩家”的核心问题。

### 1.2 设计理念

| 理念 | 说明 |
|------|------|
| **全覆盖** | 任何玩家入驻后，至少能找到2-3个“属于我”的俱乐部 |
| **可生长** | 俱乐部不依赖运营手动创建，玩家提议+系统自动审核形成正循环 |
| **有竞争** | 俱乐部活力值体系，让俱乐部之间形成良性竞争，避免死群 |
| **可筛选** | 兴趣标签让玩家在海量俱乐部中精准找到“我的同类” |

### 1.3 与《PRD-V1.0》俱乐部章节的关系

| 层级 | 覆盖内容 | 关系 |
|------|----------|------|
| 《PRD-V1.0-俱乐部系统》 | 俱乐部列表、详情页、发帖/回帖 | **底层**：定义俱乐部的基础功能 |
| **本文档（俱乐部升级系统）** | 俱乐部类型扩展、自定义提议、标签系统、活力值机制 | **上层**：在基础功能之上叠加“增长引擎” |

**两者是叠加关系**——V1.0实施时可直接采用本文档的四层俱乐部结构，替代PRD中的简单俱乐部列表。


## 二、俱乐部体系结构

### 2.1 四层俱乐部体系

| 层级 | 名称 | 数量 | 创建方式 | 加入方式 | 覆盖目标 |
|------|------|------|----------|----------|----------|
| **第一层：默认层** | 闲云茶馆 | 1个 | 系统预置 | **自动加入**（所有玩家入驻即加入） | 100%玩家有“家” |
| **第二层：兴趣层** | 演武场/农耕社/工匠坊/戏台/画师阁/酒馆/维修铺 | 7个 | 系统预置 | **自由加入**（玩家自行选择） | 覆盖主流玩家画像 |
| **第三层：游戏层** | 按具体游戏名称创建 | 首批5-8款 | 系统预置 + 动态增加 | **自由加入**（玩家自行选择） | 覆盖热门游戏玩家 |
| **第四层：长尾层** | 玩家自定义俱乐部 | 动态增长 | **玩家提议 + 系统自动/人工审核** | **自由加入 / 审批加入** | 覆盖长尾需求 |

### 2.2 覆盖度目标

| 目标 | 说明 | 验证方式 |
|------|------|----------|
| **入驻即归属** | 新玩家入驻后自动进入“闲云茶馆”，0秒孤独 | AC-01 |
| **至少3个可加入** | 任何玩家在兴趣层+游戏层中，至少能找到3个感兴趣的俱乐部 | AC-02 |
| **长尾自治** | 玩家提议的新俱乐部，在达到门槛后自动开放，无需运营介入 | AC-03 |
| **90%覆盖** | 90%的玩家至少在一个活跃俱乐部中 | AC-04 |


## 三、详细功能设计

### 3.1 默认俱乐部：“闲云茶馆”

**定位**：所有玩家的“第一个家”。不求深度，只求存在。

**配置**：

| 属性 | 值 |
|------|-----|
| **名称** | 闲云茶馆 |
| **古风包装** | “南来北往的客官，进来喝杯茶再走。” |
| **类型** | `default` |
| **创建方式** | 系统预置，开服即存在 |
| **加入方式** | `auto`（所有玩家入驻即自动加入，不可退出） |
| **话题范围** | 任何话题（水帖、闲聊、问路、吐槽） |
| **特殊规则** | 允许水帖，内容审核相对宽松，仅过滤违规内容 |

**玩家体验**：
- 入驻后，聊天列表自动出现“🍵 闲云茶馆”群聊
- 守护灵发送入群欢迎消息：“这里是闲云茶馆。累了就进来坐坐，不用说话也没关系。”
- 玩家可在此发起任何话题，无需匹配俱乐部主题

### 3.2 兴趣俱乐部（7个）

**定位**：按玩家属性/兴趣聚合，不依赖具体游戏，确保覆盖所有玩家类型。

**完整列表**：

| 古风名称 | 兴趣标签 | 覆盖人群 | 典型话题 |
|----------|----------|----------|----------|
| **⚔️ 演武场** | `pvp`, `竞技`, `对战` | 喜欢PVP、竞技、天梯排名的玩家 | 英雄强度、上分攻略、BP策略 |
| **🌾 农耕社** | `休闲`, `收集`, `种田` | 喜欢种田、收集、养成、轻松的玩家 | 作物布局、收集进度、佛系玩法 |
| **🏗️ 工匠坊** | `建造`, `经营`, `策略` | 喜欢建造、经营、策略规划的玩家 | 基地设计、资源规划、布局灵感 |
| **🎭 戏台** | `剧情`, `RPG`, `二次元` | 喜欢剧情、角色扮演、二次元的玩家 | 剧情讨论、角色厨放、同人创作 |
| **📷 画师阁** | `创作`, `截图`, `美术` | 喜欢分享截图、创作、美术的玩家 | 截图分享、绘画交流、滤镜安利 |
| **🎙️ 酒馆** | `社交`, `聊天`, `交友` | 喜欢聊天吹水、交友的玩家 | 日常闲聊、组队邀请、扩列交友 |
| **🔧 维修铺** | `数码`, `硬件`, `技术` | 数码爱好者、硬件玩家（你的老本行） | 配置讨论、帧率优化、外设推荐 |

**玩家体验**：
- 入驻后，俱乐部列表页默认显示“你可能感兴趣的”兴趣俱乐部推荐（基于入驻时选择的游戏偏好+守护灵类型）
- 玩家可自由加入任意兴趣俱乐部，无数量限制
- 每个兴趣俱乐部的详情页展示该俱乐部的兴趣标签，方便玩家理解话题范围

### 3.3 游戏俱乐部（动态增长）

**定位**：按具体游戏名称创建，是玩家最核心的“圈子”。

| 属性 | 说明 |
|------|------|
| **名称格式** | 《[游戏名称]》茶摊（如“《原神》茶摊”） |
| **创建方式** | 系统预置首批热门游戏（5-8款）+ 玩家提议动态增加 |
| **加入方式** | `free`（自由加入，无需审批） |
| **话题范围** | 仅限该游戏相关内容 |

**首批预置游戏**（示例，实际根据市场热度确定）：
1. 《原神》
2. 《王者荣耀》
3. 《金铲铲之战》
4. 《崩坏：星穹铁道》
5. 《绝区零》
6. 《永劫无间》手游
7. 《蛋仔派对》
8. 《和平精英》

**游戏俱乐部与提议机制的衔接**（详见3.4）：
- 当玩家提议的俱乐部名称符合“某款游戏”格式，且该游戏尚未存在俱乐部时，触发游戏俱乐部创建流程
- 后台收到“游戏类”提议后，进入较快的自动审批通道（如3天内自动批准）

### 3.4 长尾层：玩家提议俱乐部（自定义）

**定位**：覆盖“官方没想到”的长尾需求，让玩家成为俱乐部的“创建者”。

**提议流程**：

```
步骤1：玩家进入俱乐部列表页
    ↓
步骤2：点击“提议新茶摊”按钮
    ↓
步骤3：填写表单
    - 俱乐部名称（必填，限15字，古风包装友好）
    - 一句话简介（必填，限50字）
    - 所属类型（选填：游戏/兴趣/其他）
    - 关联的游戏名称（选填）
    - 兴趣标签（选填，最多3个）
    ↓
步骤4：提交 → 进入“审核中”状态
    ↓
步骤5：后台自动/人工审核
    ├── 自动批准条件：
    │   ├── 俱乐部类型为“游戏类”，且该游戏已有≥50个玩家入驻GamDen
    │   └── 或 支持该提议的玩家数 ≥ 20（通过点赞/联署机制）
    ├── 人工审核条件：
    │   ├── 名称包含敏感词 → 驳回
    │   └── 话题范围与本平台定位不符 → 驳回
    └── 运营手动批准/驳回
    ↓
步骤6：审核通过 → 新俱乐部上线
    ↓
步骤7：提议人获得奖励
    - 金币 +50
    - 守护灵亲密度 +10
    - 系统消息：俱乐部首任“掌柜”（管理权限）
```

**审核规则表**：

| 条件 | 自动批准 | 人工审核 | 驳回 |
|------|---------|---------|------|
| 名称含敏感词 | | | ✅ |
| 该游戏已有≥50个玩家入驻 | ✅ | | |
| 联署数 ≥ 20 | ✅ | | |
| 提议类型为“游戏类”但游戏玩家<50 | | ✅ | |
| 话题范围与本平台定位不符 | | | ✅（由运营判断） |
| 名称与现有俱乐部重复 | | | ✅ |
| 正常提议 | | ✅（3个工作日内处理） | |

**联署机制**（助力自动审批）：
- 提议提交后，生成一个“联署页面”，其他玩家可点击“支持”
- 联署数达到20 → 自动触发审核 → 自动批准 → 俱乐部上线
- 联署有效期7天，过期未达标则提议失效
- 联署页面展示于俱乐部列表页的“热门提议”区域

### 3.5 俱乐部兴趣标签系统

**定位**：让玩家在海量俱乐部中快速找到“我的同类”。

| **玩法偏好** | `play_style` | PVP / PVE / 收集 / 建造 / 剧情 / 社交 | 运营预置 |
| **玩家属性** | `player_type` | 硬核 / 休闲 / 社交达人 / 独狼 / 肝帝 | 运营预置 + 玩家加入俱乐部后系统自动分析 |

- 玩家加入的俱乐部 → 记录其标签 → 推荐同类标签的俱乐部


## 四、俱乐部活力值系统

**定位**：让俱乐部之间形成良性竞争，避免“死群”，用数据驱动运营。

### 4.1 活力值计算规则

| 维度 | 计分规则 | 每日上限 |
|------|----------|----------|
| **新成员加入** | +10/人 | 50 |
| **新帖发布** | +2/帖 | 100 |
| **帖子互动**（点赞/回复） | +1/次 | 80 |
| **今日活跃用户数**（发帖/回帖） | +5/人 | 50 |
| **连续活跃天数**（≥10帖/日） | +20/天 | 不限 |
| **新人留存率**（7天内再次发帖） | +30/人 | 不限 |

### 4.2 活力值等级

| 等级 | 活力值范围 | 标识 | 特权（V2.0） |
|------|-----------|------|-------------|
| **青铜** | 0-199 | 🥉 | 基础功能 |
| **白银** | 200-499 | 🥈 | 俱乐部列表排序靠前 |
| **黄金** | 500-999 | 🥇 | 列表页置顶+专属徽章 |
| **钻石** | 1000+ | 💎 | 展会优先推荐位 + 线下活动优先支持 |

### 4.3 活力值展示

- 俱乐部列表页：每个俱乐部卡片显示活力值等级徽章（🥉🥈🥇💎）+ 今日活力值变化（如“↑23”）
- 俱乐部详情页顶部：显示活力值等级、本周趋势
- 俱乐部排行榜：按活力值排序，每周更新（入口：俱乐部列表页 → “🏆 本周热门茶摊”）

### 4.4 低活力俱乐部处理

| 条件 | 处理方式 |
|------|----------|
| 连续7天活力值=0 | 系统发送提醒给俱乐部“掌柜”（若存在） |
| 连续14天活力值=0 | 俱乐部进入“休眠”状态，从列表页隐藏（但仍可通过搜索找到） |
| 连续30天活力值=0 | 自动归档，提议人可以重新激活 |

> **注意**：默认俱乐部“闲云茶馆”不参与活力值排名，始终置顶。


## 五、俱乐部与OpenIM的集成

### 5.1 俱乐部 = OpenIM群组

- 每个俱乐部对应一个OpenIM群组，群组ID与俱乐部ID一一对应
- 玩家加入俱乐部 → 自动加入OpenIM群组
- 玩家退出俱乐部 → 自动退出OpenIM群组
- 俱乐部的“帖子”发布在对应的OpenIM群组中

### 5.2 俱乐部类型与OpenIM群组类型的映射

| 俱乐部类型 | OpenIM群组类型 | 说明 |
|-----------|---------------|------|
| `default`（闲云茶馆） | 公开群 | 所有玩家自动加入，不可退出 |
| `interest` | 公开群 | 玩家自由加入 |
| `game` | 公开群 | 玩家自由加入 |
| `custom`（玩家提议） | 公开群或私有群 | 根据提议时选择的“加入方式”决定 |

### 5.3 俱乐部数据的OpenIM扩展字段

在OpenIM群组的自定义字段中存储俱乐部相关数据：

```json
{
  "club_type": "interest",
  "tags": ["pvp", "竞技"],
  "vitality": 567,
  "vitality_level": "gold",
  "owner_id": "user_xxx"  // 俱乐部的“掌柜”
}
```

## 七、数据模型

### 7.1 俱乐部表 `clubs`（扩展）

```sql
CREATE TABLE clubs (
  id PRIMARY KEY,
  -- 基础信息
  name VARCHAR(50) NOT NULL,
  description VARCHAR(200),
  -- 俱乐部类型
  club_type ENUM('default', 'interest', 'game', 'custom') NOT NULL,
  game_name VARCHAR(50),  -- 仅当 club_type='game' 时填写
  -- 标签
  tags VARCHAR[] DEFAULT '{}',
  -- 加入方式
  join_type ENUM('auto', 'free', 'approval') DEFAULT 'free',
  -- 状态
  status ENUM('active', 'dormant', 'archived') DEFAULT 'active',
  -- 成员数据
  member_count INTEGER DEFAULT 0,
  post_count INTEGER DEFAULT 0,
  -- 活力值
  vitality INTEGER DEFAULT 0,
  vitality_level ENUM('bronze', 'silver', 'gold', 'diamond') DEFAULT 'bronze',
  vitality_updated_at TIMESTAMP DEFAULT NOW(),
  -- OpenIM群组关联
  group_id VARCHAR(50) NOT NULL,
  -- 时间戳
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);
```
