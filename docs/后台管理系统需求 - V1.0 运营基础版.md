## 1. 系统定位与核心功能

### 1.1 系统定位
后台管理系统是GamDen V1.0社区的运营支撑平台，为运营人员提供用户管理、内容审核、数据监控、配置管理等核心能力。V1.0暂不包含游戏联运相关模块（游戏管理、订单管理、分成结算）。

### 1.2 核心功能模块

| 模块 | 功能子项 | V1.0实现范围 | V2.0规划 |
|------|----------|--------------|----------|
| **用户管理** | 查看用户列表、手动发放邀请码、冻结/解冻账号、查看邀请关系链 | ✅ 完整实现 | - |
| **领地与地图监控** | 查看全局地图热力图、手动调整用户坐标、配置野兽潮参数 | ✅ 完整实现 | - |
| **内容审核** | 待审核池、AI初审+人工复审、敏感词库管理、违规处理 | ✅ 完整实现 | - |
| **俱乐部管理** | 创建/关闭俱乐部、指定管理员、查看活跃度 | ✅ 完整实现 | 排行榜、香火值 |
| **数据看板** | DAU、新增用户、留存率、K因子、等级分布、小程序转化 | ✅ 完整实现 | - |
| **系统配置** | 敏感词库、野兽潮参数、金币奖励规则 | ✅ 完整实现 | - |

## 2. 用户管理模块

### 2.1 用户列表

**功能描述**：运营人员查看所有注册用户的基本信息，支持筛选、搜索、导出。

**字段清单**：

| 字段名 | 说明 | 是否可筛选 | 是否可排序 |
|--------|------|------------|------------|
| ID | 用户ID（bigint） | ✅ | ✅ |
| 手机号 | 脱敏显示（138****1234） | ❌ | ❌ |
| 守护灵类型 | 机械师/精灵/占星师 | ✅ | ❌ |
| 等级 | 领地等级（Lv.1~Lv.5） | ✅ | ✅ |
| 经验值 | 当前经验值 | ❌ | ✅ |
| 领地坐标 | (X, Y) | ❌ | ❌ |
| 金币余额 | 当前金币 | ❌ | ✅ |
| 邀请码 | 个人邀请码 | ✅ | ❌ |
| 邀请人数 | 已成功邀请人数（is_active=true） | ❌ | ✅ |
| 注册时间 | 注册时间戳 | ✅ | ✅ |
| 最后登录时间 | 最后登录时间戳 | ✅ | ✅ |
| 状态 | active/frozen/deleted | ✅ | ❌ |
| 角色 | player/vip/club_admin/operator/super_admin | ✅ | ❌ |

**筛选条件**：
- 守护灵类型：多选（机械师/精灵/占星师）
- 等级范围：Lv.1~Lv.5
- 注册时间范围：日期选择器
- 状态：单选（active/frozen/deleted）
- 角色：多选

**搜索功能**：
- 支持按用户ID、手机号（脱敏）、邀请码搜索

**批量操作**：
- 批量冻结：选择多个用户，填写原因，执行冻结
- 批量发放邀请码：选择多个用户，生成邀请码并通知用户

**导出功能**：
- 导出为Excel（.xlsx），包含当前筛选结果的所有字段

### 2.2 手动发放邀请码

**功能描述**：用于活动或KOL合作，运营人员手动为用户生成邀请码。

**操作流程**：
1. 运营人员在用户列表选择目标用户（或输入用户ID）
2. 点击"发放邀请码"
3. 系统生成6位字母数字邀请码（如：A3B7C9）
4. 邀请码绑定该用户ID，有效期30天
5. 系统自动发送短信/邮件通知用户

**关键规则**：
- 一个用户只能有一个有效邀请码（如已有邀请码且未使用，不可重复生成）
- 邀请码有效期：30天，过期自动失效
- 邀请码使用后立即失效（不可重复使用）

### 2.3 冻结/解冻账号

**功能描述**：处理违规用户，冻结后该用户无法登录。

**冻结操作**：
1. 运营人员选择目标用户
2. 点击"冻结账号"
3. 填写冻结原因（必填，如：发布违规内容）
4. 选择冻结时长：24小时/7天/永久
5. 执行冻结，系统记录操作日志

**解冻操作**：
1. 运营人员在"已冻结用户"列表选择目标用户
2. 点击"解冻账号"
3. 填写解冻原因（必填）
4. 执行解冻，系统记录操作日志

**关键规则**：
- 冻结后用户尝试登录时，提示"账号已被冻结，原因：XXX"
- 永久冻结的账号，需用户申诉后由运营人工复核
- 所有冻结/解冻操作必须记录日志（operator_id, action, target_user_id, reason, timestamp）

### 2.4 查看邀请关系链

**功能描述**：查看用户的完整邀请链，用于分析邀请裂变效果。

**展示方式**：
- 树状图：以当前用户为根节点，展示其邀请的所有用户，以及这些用户再邀请的用户
- 列表视图：表格形式展示（用户ID、被邀请人ID、邀请时间、是否活跃）

**数据来源**：`invite_records`表

**关键字段**：
- 邀请人ID、被邀请人ID、邀请时间、是否活跃（is_active）
- 如is_active=false，显示"待激活（需活跃7天）"

## 3. 领地与地图监控模块

### 3.1 全局地图热力图

**功能描述**：可视化展示玩家密集区域，帮助运营了解用户分布。

**展示方式**：
- 热力图：颜色深浅表示玩家密度（深色=高密度）
- 坐标范围：-1000~+1000（X和Y）
- 支持缩放和拖拽

**数据来源**：`users`表的`territory_coord_x`和`territory_coord_y`字段

**筛选条件**：
- 等级范围：Lv.1~Lv.5
- 注册时间范围
- 活跃度：最近7天登录/最近30天登录

### 3.2 手动调整用户坐标

**功能描述**：处理异常情况（如坐标分配错误、用户反馈坐标不合理）。

**操作流程**：
1. 运营人员输入用户ID或昵称，查询目标用户
2. 查看当前坐标和邻居信息
3. 输入新坐标（X, Y）
4. 系统检查新坐标是否已被占用
5. 如未被占用，执行更新；如已被占用，提示"坐标已被占用，请重新输入"
6. 执行更新，系统记录操作日志

**关键规则**：
- 调整后，该用户的邻居关系会发生变化（重新计算±10格范围内的玩家）
- 所有坐标调整操作必须记录日志（operator_id, target_user_id, old_coord, new_coord, reason, timestamp）
- 仅超级管理员可调整坐标，普通运营人员无权限

### 3.3 配置野兽潮刷新参数

**功能描述**：运营人员配置野兽潮的刷新时间、频率、强度。

**配置项**：

| 配置项 | 说明 | 默认值 | 取值范围 |
|--------|------|--------|----------|
| 刷新时间间隔 | 多久检查一次是否触发野兽潮 | 1小时 | 1~24小时 |
| 触发概率 | 每次检查时触发野兽潮的概率 | 20% | 0%~100% |
| 强度范围 | 野兽潮等级（1~5） | 1~3 | 1~5 |
| 影响范围 | 野兽潮影响周围多少格 | 20格 | 10~50格 |
| 防御失败概率 | Lv.3以下领地防御失败的概率 | 30% | 0%~100% |

**操作流程**：
1. 运营人员进入"野兽潮配置"页面
2. 修改配置项
3. 点击"保存配置"
4. 系统更新`beast_config`表
5. 配置立即生效（无需重启服务）

**关键规则**：
- 配置变更必须记录日志（operator_id, old_config, new_config, timestamp）
- 仅超级管理员可修改配置，普通运营人员仅可查看

## 4. 内容审核模块

### 4.1 待审核池

**功能描述**：展示所有待人工复审的内容（AI初审不确定或标记为高风险）。

**内容类型**：

| 内容类型 | 字符限制 | AI初审规则 | 是否进入人工复审池 |
|----------|----------|------------|---------------------|
| 用户签名 | ≤20字符 | 置信度>90%直接通过，≤90%进入人工复审 | 是（置信度≤90%） |
| 俱乐部帖子 | ≤500字符 | 同上 | 是（置信度≤90%或敏感） |
| 回帖 | ≤200字符 | 同上 | 仅高风险（置信度≤70%） |

**展示方式**：
- 列表视图：表格形式展示（内容ID、用户ID、内容类型、内容摘要、AI初审结果、提交时间、状态）
- 详情视图：点击某条内容，展示完整内容、AI初审详情（置信度、命中敏感词）、操作按钮（通过/拒绝）

**筛选条件**：
- 内容类型：签名/帖子/回帖
- 状态：pending/approved/rejected
- 提交时间范围
- AI初审结果：通过/不确定/拒绝

### 4.2 AI初审+人工复审流程

**流程图**：

```
[用户] 发布内容（签名/发帖/回帖）
    ↓
[系统] 调用腾讯云内容安全API（AI初审）
    ↓
AI初审结果？
    ↓                    ↓                    ↓
通过（>90%）          不确定（70%~90%）      拒绝（<70%或敏感）
    ↓                    ↓                    ↓
[系统] 内容发布成功    [系统] 进入人工复审池    [系统] 拒绝发布，通知用户
    ↓                    ↓
[用户] 内容可见        [运营] 人工审核
                         ↓
            审核通过？           审核拒绝？
             ↓                    ↓
[系统] 内容发布成功    [系统] 通知用户+记录违规
```

**操作按钮**：
- 通过：内容发布成功，通知用户
- 拒绝：填写拒绝原因（必填），通知用户，记录违规
- 查看AI初审详情：显示置信度、命中敏感词、建议操作

### 4.3 敏感词库管理

**功能描述**：运营人员管理敏感词库，支持添加、删除、导入、导出。

**敏感词分类**：

| 分类 | 说明 | 示例 |
|------|------|------|
| 政治敏感 | 政治人物、政治事件 | - |
| 色情 | 色情词汇 | - |
| 暴力 | 暴力、恐怖词汇 | - |
| 人身攻击 | 辱骂、歧视词汇 | - |
| 商业广告 | 广告、推广词汇 | - |

**操作功能**：
- 添加敏感词：输入敏感词、选择分类、设置级别（高/中/低）
- 删除敏感词：选择敏感词，执行删除
- 批量导入：上传Excel文件（.xlsx），批量导入敏感词
- 批量导出：导出为Excel文件

**关键规则**：
- 敏感词库用于AI内容安全API的自定义词库（如腾讯云支持自定义词库）
- 敏感词级别：高级别（直接拒绝）、中级别（进入人工复审）、低级别（标记警告）

### 4.4 违规处理梯度

**功能描述**：根据用户违规次数，自动执行梯度处罚。

**梯度规则**：

| 违规次数 | 处罚措施 | 执行方式 |
|----------|----------|----------|
| 第一次 | 警告 | 系统自动发送警告通知 |
| 第二次 | 禁言24小时 | 系统自动执行，用户无法发布内容 |
| 第三次 | 冻结账号 | 系统自动执行，需运营人工复核 |

**操作流程**：
1. 运营人员审核拒绝某条内容时，选择"记录违规"
2. 系统自动累加该用户的违规次数（user_violations表）
3. 根据违规次数，自动执行梯度处罚
4. 通知用户处罚结果

**申诉机制**：
- 用户收到处罚通知后，可点击"申诉"
- 系统生成申诉工单（appeals表）
- 运营人员在后台查看申诉工单，人工复核
- 如复核通过，撤销处罚；如复核拒绝，维持处罚

## 5. 俱乐部管理模块

### 5.1 创建/关闭俱乐部

**创建俱乐部**：
- 前端：Lv.2以上用户可创建俱乐部
- 后台：运营人员也可创建俱乐部（用于官方运营）

**关闭俱乐部**：
- 运营人员选择目标俱乐部
- 点击"关闭俱乐部"
- 填写关闭原因（必填）
- 执行关闭，俱乐部状态改为"closed"
- 俱乐部内所有帖子隐藏（不删除，仅隐藏）
- 通知俱乐部管理员

**关键规则**：
- 关闭后的俱乐部不可重新开启（如需重新开启，需重新创建）
- 关闭操作必须记录日志（operator_id, club_id, reason, timestamp）

### 5.2 指定俱乐部管理员

**功能描述**：运营人员指定某用户为俱乐部管理员。

**操作流程**：
1. 运营人员选择目标俱乐部
2. 输入用户ID或昵称，查询目标用户
3. 点击"指定为管理员"
4. 系统更新`clubs`表的`owner_id`字段
5. 通知该用户

**关键规则**：
- 俱乐部管理员可以删除俱乐部内的帖子、管理成员
- 一个俱乐部只能有一个管理员（owner_id）
- 指定管理员操作必须记录日志

### 5.3 查看俱乐部活跃度

**功能描述**：展示俱乐部的活跃度指标，帮助运营了解俱乐部健康状况。

**活跃度指标**：

| 指标 | 说明 | 统计口径 |
|------|------|----------|
| 成员数 | 俱乐部总成员数 | `clubs`表的`member_count`字段 |
| 帖子数 | 俱乐部总帖子数 | `clubs`表的`post_count`字段 |
| 发帖数（近7天） | 最近7天发布的帖子数 | 统计`club_posts`表，created_at≥7天前 |
| 回帖数（近7天） | 最近7天发布的回帖数 | 统计`club_replies`表，created_at≥7天前 |
| 最后活跃时间 | 俱乐部最后发帖/回帖时间 | `clubs`表的`last_active_at`字段 |

**展示方式**：
- 列表视图：表格形式展示所有俱乐部的活跃度指标
- 详情视图：点击某个俱乐部，展示详细数据和趋势图（近30天发帖趋势）

**筛选条件**：
- 活跃度范围：高/中/低（根据发帖数近7天自动判定）
- 最后活跃时间范围

## 6. 数据看板模块

### 6.1 关键指标定义

| 指标 | 说明 | 统计口径 | 展示方式 |
|------|------|----------|----------|
| **日活跃用户（DAU）** | 当日登录的用户数 | 统计`users`表，last_login_at=今日 | 折线图（近30天） |
| **新增用户** | 当日注册的用户数 | 统计`users`表，created_at=今日 | 折线图（近30天） |
| **次日留存率** | 昨日注册用户中，今日仍登录的比例 | 统计`invite_records`表（需补充留存率计算逻辑） | 折线图（近30天） |
| **邀请裂变K因子** | 平均每个用户邀请多少人 | 总邀请人数 / 总注册人数 | 数字卡片 |
| **领地等级分布** | 各等级用户占比 | 统计`users`表，按level分组 | 饼图 |
| **金币流通量** | 系统中总金币数量 | 统计`users`表的gold_coins字段总和 | 折线图（近30天） |
| **小程序访问量** | 小程序每日访问次数 | 需接入小程序数据分析 | 折线图（近30天） |
| **App下载转化率** | 从小程序下载App的比例 | 需接入下载追踪 | 数字卡片 |

### 6.2 数据展示方式

**总览页**：
- 数字卡片：DAU、新增用户、K因子、金币流通量
- 折线图：DAU趋势（近30天）、新增用户趋势（近30天）
- 饼图：领地等级分布

**用户分析页**：
- 折线图：注册用户趋势（近30天）、留存率趋势（近30天）
- 表格：用户列表（同用户管理模块）

**邀请分析页**：
- 数字卡片：K因子、平均邀请人数
- 折线图：邀请人数趋势（近30天）
- 表格：邀请关系链（同用户管理模块）

**内容分析页**：
- 数字卡片：今日发帖数、今日回帖数
- 折线图：发帖趋势（近30天）
- 表格：热门帖子排行（按like_count排序）

### 7.3 金币奖励规则配置

**功能描述**：运营人员配置金币获取和消耗的规则。

**配置项**：

| 配置项 | 说明 | 默认值 | 取值范围 |
|--------|------|--------|----------|
| 注册奖励 | 新用户注册获得的金币 | 100 | 0~1000 |
| 每日签到奖励 | 每日签到获得的金币 | 10 | 0~100 |
| 连续签到额外奖励 | 连续签到每天额外获得的金币 | 5 | 0~50 |
| 发帖奖励 | 每发一帖获得的金币 | 5 | 0~50 |
| 回帖奖励 | 每回一帖获得的金币 | 2 | 0~20 |
| 邀请好友奖励 | 每邀请一人获得的金币 | 50 | 0~500 |
| 防御野兽潮奖励 | 防御成功获得的金币（按等级） | 50~200 | 0~1000 |

**操作流程**：
1. 运营人员进入"金币规则配置"页面
2. 修改配置项
3. 点击"保存配置"
4. 系统更新`gold_config`表
5. 配置立即生效

**关键规则**：
- 配置变更必须记录日志
- 仅超级管理员可修改配置

## 8. 权限管理与操作日志

### 8.1 角色权限定义

| 角色名称 | 权限 |
|----------|------|
| **运营人员（operator）** | 查看用户列表、冻结/解冻账号、内容审核、查看数据看板、查看俱乐部活跃度 |
| **超级管理员（super_admin）** | 所有权限（包括系统配置、手动调整坐标、管理运营人员） |

**权限校验规则**：
- 所有后台API请求必须在Header中携带JWT Token，服务端校验用户角色
- 前端根据角色隐藏无权限的按钮和菜单

### 8.2 操作日志

**功能描述**：记录所有敏感操作，用于审计和追溯。

**日志记录范围**：
- 用户冻结/解冻
- 坐标调整
- 内容审核（通过/拒绝）
- 俱乐部创建/关闭
- 系统配置变更
- 敏感词库管理

**日志字段**：

| 字段名 | 说明 |
|--------|------|
| operator_id | 操作人ID |
| action | 操作类型（freeze_user/adjust_coord/approve_content等） |
| target | 操作目标（用户ID/内容ID/配置项等） |
| reason | 操作原因 |
| old_value | 旧值（如坐标调整前的坐标） |
| new_value | 新值（如坐标调整后的坐标） |
| timestamp | 操作时间 |

**查看日志**：
- 运营人员可查看所有日志
- 支持按操作人、操作类型、时间范围筛选
