# GamDen 游客模式体验设计需求文档

**版本**：V1.0 | **状态**：待评审 | **关联项目**：GamDen App V1.0


## 一、设计目标

### 1.1 核心策略
**“先逛后住，降低门槛”** ——让用户在未登录状态下自由探索GamDen的世界，感受氛围和价值，自然产生注册意愿，而非被强制登录拦截在门外。

### 1.2 关键指标（用于验证效果）
- 游客→注册转化率 ≥ 25%（行业参考：优质内容App的游客转化率约20-40%）
- 游客平均浏览时长 ≥ 90秒（足以浏览3-5个页面）
- 注册流程完成率 ≥ 85%（漏斗转化）

### 1.3 设计原则
| 原则 | 说明 |
|------|------|
| **内容全可见** | 不因游客身份隐藏任何内容，只锁定交互行为 |
| **引导不打扰** | 软性提示优于强制弹窗，守护灵劝说优于系统Alert |
| **状态无感切换** | 注册后自动回到浏览位置，不丢失上下文 |
| **世界观自洽** | 所有限制提示用“守护灵对话”形式呈现，而非冷冰冰的系统提示 |

## 三、各页面游客态详细设计

### 3.1 地图页面（核心浏览入口）

#### 3.1.1 顶部状态栏
| 元素 | 游客态显示 | 说明 |
|------|-----------|------|
| 等级显示 | 隐藏 | 游客无等级概念 |
| 金币余额 | 显示“???”或“0” | 视觉上占位，不展示具体数值 |
| 未读消息红点 | 隐藏 | IM模块完全不可见 |
| 右上角“登录/注册”按钮 | 显示 | 文案为“入巢”，点击进入注册页 |

#### 3.1.2 地图画布
- **缩放与拖动**：完全开放，无限制。
- **领地显示**：显示所有已入驻玩家的领地，图标、昵称、等级**完全可见**。
- **坐标模糊化**：每个领地卡片上的坐标显示为`"附近"`或`"坐标：XX·XX"`（末位用`·`替代数字），不暴露精确位置。
- **点击邻居领地**：弹出卡片，显示该玩家信息（昵称、守护灵类型、领地等级、最近常玩游戏标签）。卡片底部按钮文案为 **“拜访TA的巢穴 →”** ，点击后弹窗引导登录。

#### 3.1.3 无人区迷雾
- 维持V1.0设计的深色迷雾 + 边缘微光动画，对游客完全可见。
- 野兽踪迹动画正常播放，激发探索欲和好奇心。

#### 3.1.4 底部Tab栏
| Tab | 游客态行为 |
|-----|-----------|
| **地图** | 可点击，当前页 |
| **俱乐部** | 可点击，进入俱乐部列表 |
| **集市** | 可点击，进入集市 |
| **我的** | **隐藏，不显示** |


### 3.2 俱乐部页面（内容浏览入口）

#### 3.2.1 俱乐部列表
- 完全展示所有俱乐部卡片（游戏名称、成员数、今日新帖数）。
- 搜索框可用，输入关键词筛选俱乐部。

#### 3.2.2 俱乐部详情页（帖子列表+帖子正文）
- **帖子列表**：完整展示所有帖子标题、作者、发布时间、预览内容。
- **帖子正文**：游客可**完整阅读**任何帖子的全部内容。
- **底部交互栏**：显示“点赞数”和“评论数”，但两个按钮**置灰不可点击**。点击时弹出引导弹窗。

#### 3.2.3 “我要发帖”按钮
- **完全隐藏**。游客无发帖入口。


### 3.3 集市页面（商业化预览入口）

#### 3.3.1 道具列表
- 完整展示所有虚拟道具（名称、描述、所需金币数、图标）。

#### 3.3.2 兑换按钮
- 所有道具的“兑换”按钮**显示为灰色不可点**，按钮文案保持“兑换”但不响应点击。
- 若游客点击灰色按钮区域，触发引导弹窗。

#### 3.3.3 金币余额
- 顶部显示为`"???"`或`"0"`，不展示精确数值。


### 3.4 引导弹窗设计（关键交互）

#### 3.4.1 触发场景
任何游客点击受限操作（点赞、回复、兑换、拜访邻居）时触发。

#### 3.4.2 弹窗设计规范
| 要素 | 规范 |
|------|------|
| **样式** | 全屏半透明蒙层 + 居中卡片，卡片背景为“宣纸白”，带古风边框纹理 |
| **头部** | 守护灵头像（根据用户尚未选择，默认显示GamDen Logo）+ 守护灵对话气泡 |
| **文案模板** | “你在外面看了那么久，不进来坐坐吗？——只需要一个名字，我就给你一块地。” |
| **按钮1（主）** | 【领取我的领地】——绿色，跳转注册页 |
| **按钮2（次）** | 【再看看】——透明无边框，关闭弹窗 |


## 四、守护灵“软引导”策略

### 4.1 触发条件
游客浏览时长 ≥ 30秒，且未进行任何注册操作。

### 4.2 表现形式
页面底部从下往上**缓慢浮起**一个圆形头像气泡（带守护灵头像），持续3秒后自动展开为一句话。

### 4.3 文案库（随机轮换，避免重复）

| 守护灵 | 文案 |
|--------|------|
| **机械师** | “检测到观望信号。巢穴的门是开着的，随时欢迎归队。” |
| **精灵** | “你站在领地边缘很久了。风在等你做出决定。” |
| **占星师** | “星象显示，你即将做出一个重要的选择。不急，我等你。” |

### 4.4 交互
- 点击气泡 → 展开为完整引导弹窗（同3.4.2）。
- 不点击 → 15秒后自动淡出消失。


## 五、注册后“无缝归巢”流程

### 5.1 触发
游客点击任何引导弹窗的【领取我的领地】按钮 → 进入注册页。

### 5.2 注册页设计
| 元素 | 规范 |
|------|------|
| 头部 | 守护灵对话气泡：“告诉我你的名字，这块地就是你的了。” |
| 输入项 | 仅**手机号 + 验证码**（或微信一键登录），不要求填写昵称/头像 |
| 按钮 | 【归巢】——主品牌色 |
| 协议 | 底部小字：《用户协议》和《隐私政策》 |

### 5.3 注册成功后的过渡动画
1. 注册完成 → 屏幕暗转。
2. 播放“种子落地”微动画（约1.5秒）——一粒光点从屏幕中央落下，落地后扩散出一圈涟漪。
3. 涟漪散去 → **自动进入地图页面（登录态）**，地图中心显示用户刚获得的领地，系统Toast提示：“欢迎归巢，第XXXXX号巢民。”
4. **上下文保留**：如果游客之前在浏览某个俱乐部帖子，注册成功后**直接跳转回该帖子**，并自动刷新为登录态（点赞/回复按钮变为可用）。


## 六、技术实现要求

### 6.1 游客身份标识
- App首次启动时，生成一个**设备唯一标识（DeviceID）**，存储在本地Keychain/SharedPreferences中。
- 所有API请求携带`device_id`字段，服务端据此识别游客身份。
- 游客注册后，`device_id`与新建的`user_id`做**绑定关联**，保留该设备上的浏览历史（如有）。

### 6.2 接口身份识别
- 所有接口增加`identity_type`字段（`guest` / `user`）。
- 当`identity_type=guest`时：
  - 返回数据中所有交互字段（`can_like`、`can_comment`、`can_exchange`等）均设为`false`。
  - 金币类字段返回`null`或`0`。
  - 私聊类接口直接返回`401 Unauthorized`。

### 6.3 状态管理（前端）
- 全局维护一个`AuthStore`（或类似的状态管理），包含`isLoggedIn`、`deviceId`、`userInfo`。
- 所有页面的UI渲染依赖`isLoggedIn`状态决定展示“完整版”或“预览版”。
- 引导弹窗的展示频次由前端控制：**同一设备，30分钟内最多弹出2次**，避免骚扰。

### 6.4 数据合并（注册时）
- 注册成功后，服务端执行：
  1. 将`device_id`关联的浏览日志（如浏览过的帖子ID、停留时长）迁移到`user_id`下。
  2. 清理`device_id`的游客缓存数据。
- 前端执行：清除本地游客状态，切换为登录态，重新拉取用户数据。


## 七、数据埋点要求

以下埋点用于评估游客模式效果：

| 事件名称 | 触发条件 | 附加字段 |
|---------|---------|----------|
| `guest_launch` | App首次启动（游客态） | `device_id` |
| `guest_browse_duration` | 游客浏览每30秒上报一次 | `duration_seconds`, `current_page` |
| `guest_click_restricted` | 游客点击受限操作（点赞/回复/兑换/拜访） | `action_type`, `page_name` |
| `guest_guide_shown` | 守护灵软引导弹出 | `guide_type`（软引导/硬弹窗） |
| `guest_register_click` | 游客点击注册按钮 | `source`（软引导/受限操作弹窗/主动点击） |
| `register_complete` | 注册完成 | `device_id → user_id` 映射 |


## 八、UI设计规范（游客态特有）

### 8.1 受限操作视觉反馈
| 元素 | 样式规范 |
|------|----------|
| 不可点击按钮 | 透明度 0.5，无点击反馈（无涟漪、无震动） |
| 受限操作区域 | 点击无高亮，仅触发引导弹窗 |
| 状态提示 | 不用Toast，统一用守护灵弹窗 |

### 8.2 登录入口位置
| 页面 | 入口位置 |
|------|----------|
| 地图 | 右上角“入巢”文字按钮 |
| 俱乐部 | 帖子底部受限按钮点击触发 |
| 集市 | 兑换按钮点击触发 |
| 全局 | 守护灵软引导气泡 |

## 十、附录：与主需求文档的关联

本需求文档是对《PRD-V1.0》和《UI/UX设计规范》的补充，所有游客态设计均遵循：
- 色彩系统与全局UI风格保持一致。
- 守护灵话术库（固定模板）与注册用户共用，仅触发场景不同。
- 埋点数据归入后台数据看板的“用户增长”板块。
