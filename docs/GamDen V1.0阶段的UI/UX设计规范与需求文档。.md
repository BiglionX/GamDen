这是GamDen V1.0阶段的**UI/UX设计规范与需求文档**。基于我们之前确认的"先做社区、后做联运"策略，这套规范专注于将"古风巢穴"的视觉语言与邀请裂变的核心玩法融合，并为未来引入AI Agent留下扩展接口。

---

## 一、设计理念与品牌调性

### 核心理念
GamDen的UI需要让用户产生**"进入巢穴"**的直觉感受——切换App的瞬间，仿佛从嘈杂的社交广场走进了一个只属于自己的安静角落。

**品牌关键词**：庇护感、像素古风、玩家归属、细节温度

### 视觉情绪板
- **色调**：暗金（#C9A87C）为点缀，墨绿（#2C3A2F）为主背景，米白（#F5F0E6）为文字底色——暗色调营造"巢穴"的包裹感，金色传递"宝藏/领地"的价值暗示。
- **质感**：轻微纸张纹理叠加于卡片背景，像素风格的装饰性边框，模拟"古风卷轴"的视觉触感。
- **动效原则**：所有切换操作都带微反馈（点击时涟漪、切换时淡入），但克制使用大动画——"安静"是第一优先级。

### 与OpenIM视觉集成的策略
OpenIM的UI皮肤包允许全局定制主题色、气泡样式、聊天气泡背景图。我们的策略是：**将OpenIM的IM模块完全融入GamDen的整体视觉体系，而非让其看起来像一个独立的聊天插件**。
- **主题色统一**：将OpenIM的`Global-TintColor`设为GamDen的主色调（暗金）。
- **聊天背景定制**：替换`talk_bg.jpeg`为GamDen的巢穴暗纹背景。
- **气泡样式**：在`wxo_bubble_customize.plist`中，将消息气泡背景图替换为带有古风边框纹理的样式，左右气泡颜色区分（左浅灰，右暗金）。

## 三、核心页面交互设计

### 3.1 地图（巢穴俯视图）——V1.0核心页面
这是GamDen的"主舞台"，承载领地探索与邀请裂变的直觉反馈。

**布局结构**：
- **顶部状态栏**：显示当前等级、金币余额、未读消息红点（融合OpenIM会话列表入口）。
- **中央画布**：2D网格地图，以用户领地为中心，显示周围邻居的领地图标。每个领地显示玩家头像缩略图和领地等级。
- **底部操作栏**（四个Tab）：**地图**（当前页）、**俱乐部**、**集市**、**我的**

**交互细节**：
- **双指缩放**：放大地图看到更精细的领地信息（像素图标→小屋简笔画）。
- **点击邻居领地**：弹出半透明卡片，显示该玩家的**游戏标签**（最近常玩游戏）和**打招呼**按钮——点击后进入OpenIM私聊界面，但整个聊天窗口采用GamDen的皮肤。
- **无人区迷雾**：未探索的格子以深色迷雾覆盖，边缘微光暗示"可能有野兽出没"（为V2.0野兽玩法预留视觉钩子）。

### 3.2 俱乐部（游戏贴吧）
即OpenIM群组功能在GamDen中的落地形式。每个俱乐部对应一个OpenIM群组。

**列表页**（入口）：
- 以"古风茶摊"的视觉语言展示俱乐部列表。
- 每个俱乐部卡片显示：游戏名称、成员数、今日新帖数。
- **搜索框**：输入游戏名称快速筛选（支持模糊搜索）。

**详情页**（群聊界面）：
- 沿用OpenIM的聊天界面框架，但替换所有UI元素以匹配GamDen的视觉风格。
- **顶部**：返回箭头、俱乐部名称、成员数。
- **聊天区**：消息气泡采用古风边框纹理；Agent（守护灵）在群内发言时，头像带特殊光效，气泡颜色异于普通用户。
- **输入区**：支持文字、表情（OpenIM默认表情包可保留，后续可替换为定制古风表情）。

### 3.3 集市（百货商城——V1.0仅虚拟道具）
**页面结构**：
- **顶部轮播Banner**：展示当前限时兑换活动。
- **商品列表**：以网格卡片展示虚拟道具（头像框、领地装饰、升级加速券），每张卡片显示道具名称、所需金币数、兑换按钮。
- **兑换流程**：点击"兑换"→弹出确认窗→消耗金币→道具即时到账（无须支付流程，V1.0仅消耗游戏内货币）。

### 3.4 "我的"（个人中心）
**包含模块**：
- **用户信息卡片**：头像、昵称、守护灵类型、领地等级、领地坐标（X,Y）。
- **邀请码入口**：显示当前邀请人数 + 进度条（3人解锁小程序），点击生成带参数的邀请海报。
- **个人小程序码**：邀请满3人后，此处出现"我的领地小程序码"，可保存分享。
- **设置**：退出登录、隐私设置（是否对外显示游戏标签）。

## 四、关键流程交互设计

### 4.1 新用户入驻引导（Onboarding）
**流程**：
1. **启动页**：GamDen Logo + 主文案"在算法之外，建一座游戏巢穴"。
2. **守护灵选择**：三选一（机械师/精灵/占星师），每个选项配一句简短台词和图标，让用户感知性格差异。
3. **领地分配**：全屏地图动画，一枚"种子"从空中落下，落在某个随机坐标，伴随轻微震动反馈，系统弹窗："欢迎归巢，你获得了第XXXXX号领地"。
4. **邀请引导**：在用户第一次进入地图后，Agent以消息气泡形式弹出："隔壁还是荒地，要不要拉个朋友来陪你？"

### 4.2 邀请裂变闭环
**触发**：用户点击个人中心的"邀请好友" → 生成带用户ID参数的邀请海报（或小程序码）。
**传播**：用户将海报分享到微信/Facebook。
**转化**：
1. 好友扫描海报 → 进入**营销H5**（展示GamDen世界预览 + 填写申请表单）。
2. 表单提交 → 运营后台收到申请 → 手动/自动发放邀请码。
3. 好友用邀请码注册 → 系统检测到`invited_by`字段 → 好友领地自动分配在邀请人领地**相邻格**。
4. 邀请人收到OpenIM系统通知："你的盟友已抵达，领地边界拓展了。"

### 4.3 Agent拟人化触发场景（V1.0固定话术）
| 触发事件 | Agent回复 | 展示形式 |
|---------|----------|---------|
| 每日首次登录 | "检测到信号，欢迎回巢。"（机械师）/ "晨光落在你领地上了。"（精灵） | 系统通知栏（OpenIM自定义消息） |
| 领地等级提升 | "锻造炉温度上升，你变强了。" | 顶部Toast提示 |
| 邀请成功 | "新坐标已锁定，盟友就在你身旁。" | 推送通知 + 地图闪烁 |
| 连续3天未登录 | "巢穴需要你，回来看看吧。" | 推送通知 |

## 五、与OpenIM集成的UI约束

为保证V1.0开发效率，对IM模块的定制范围需明确锁定：

### 必须定制（直接影响品牌感）
- 全局主题色改为GamDen暗金（修改`Global-TintColor`）
- 聊天窗口背景图替换为巢穴暗纹（替换`talk_bg.jpeg`）
- 消息气泡样式修改：在`wxo_bubble_customize.plist`中配置气泡背景图及圆角

### 建议保留默认（降低二开成本）
- 会话列表交互逻辑（左滑删除、置顶等）
- 表情包面板（V1.0先用OpenIM默认，后续版本再替换为古风定制）
- 图片/文件发送逻辑

### 必须新增（GamDen独有的业务消息类型）
在`exampleSendCustomMessageWithConversationController:`中扩展自定义消息类型：
- **Agent消息**：带守护灵头像、特殊气泡颜色、不可回复（仅系统单向推送）。
- **野兽警报**：V2.0预留接口，V1.0仅做数据结构占位。

## 七、开发对接说明

1. **UI开发顺序**：OpenIM皮肤定制可最先完成（纯配置工作，不涉及核心逻辑），与领地系统的开发并行推进。
2. **统一组件库**：建立`GamDen-UI-Kit`，将地图网格、领地卡片、Agent气泡等封装为复用组件，避免重复造轮子。
3. **响应式适配**：V1.0仅适配手机竖屏（375pt~428pt宽度），平板和桌面端暂不考虑。
