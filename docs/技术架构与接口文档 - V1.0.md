## 2. OpenIM二改边界与实现方案

### 2.1 二改边界（明确版）

| 层级 | 是否修改 | V1.0具体内容 | 技术实现方案 |
|------|----------|--------------|--------------|
| **OpenIM核心源码** | ❌ 不修改 | 保持原版，方便后续升级 | 通过Webhook对接，不修改核心源码 |
| **Webhook中间件** | ✅ 新增 | 监听"用户注册"事件，同步创建领地 | Node.js中间件，监听OpenIM的callback |
| **自定义消息类型** | ✅ 扩展 | Agent系统通知（带守护灵头像和语气） | OpenIM自定义消息协议（CustomMessage） |
| **用户体系** | ✅ 对接 | 用户ID与业务后台保持一致 | 统一JWT Token，OpenIM使用相同userID |
| **群组** | ✅ 使用 | 每个俱乐部对应一个OpenIM群组 | 创建俱乐部时自动创建群组 |

### 2.2 Webhook对接流程

**目标**：当用户在OpenIM注册时，自动在业务后台创建领地和初始化数据。

**流程图**：

```
[OpenIM] 用户注册成功
    ↓
[OpenIM] 发送Webhook回调（POST /webhook/openim/user-register）
    ↓
[GamDen业务后台] 接收Webhook回调
    ↓
[业务后台] 解析回调数据（userID, nickname, avatar等）
    ↓
[业务后台] 调用领地服务，分配坐标、初始化领地Lv.1、金币100
    ↓
[业务后台] 返回成功响应给OpenIM
    ↓
[OpenIM] 用户可正常使用IM功能
```

**Webhook接口定义**：

```
POST /webhook/openim/user-register
Content-Type: application/json
X-OpenIM-Signature: <signature>  // 签名验证，防止伪造请求

{
  "userID": "123456",           // OpenIM用户ID（与业务后台userID一致）
  "nickname": "玩家A",          // 用户昵称
  "avatar": "https://...",      // 用户头像
  "timestamp": 1624356000,     // 时间戳
  "signature": "abc123..."      // 签名（用于验证请求合法性）
}
```

**关键规则**：
- Webhook接口必须在5秒内返回响应，否则OpenIM会重试（最多3次）
- 如领地创建失败，返回错误码，OpenIM会标记用户注册失败
- 签名验证：使用OpenIM配置的secret_key，对userID+timestamp进行HMAC-SHA256签名

### 2.3 自定义消息类型（Agent系统通知）

**目标**：Agent（守护灵）发送的通知消息，带有守护灵头像和语气，区别于普通系统通知。

**消息结构**：

```json
{
  "customType": 1001,           // 自定义消息类型（1001=Agent通知）
  "data": {
    "agentType": "mechanic",    // 守护灵类型（mechanic/elf/astrologer）
    "avatar": "https://...",    // 守护灵头像URL
    "content": "检测到新信号。欢迎归巢，编号123456。",  // 通知内容
    "timestamp": 1624356000     // 时间戳
  }
}
```

**实现步骤**：
1. 在OpenIM中注册自定义消息类型（customType=1001）
2. 业务后台发送Agent通知时，构造自定义消息结构体
3. 调用OpenIM的发送消息API，将自定义消息发送给目标用户
4. 客户端收到自定义消息后，根据customType解析并展示（显示守护灵头像和语气）

**关键规则**：
- 自定义消息不存储到OpenIM的普通消息表，需业务后台额外存储（agent_dialogues表）
- 客户端需适配自定义消息的展示样式（守护灵头像、特殊气泡）

## 3. 核心数据表结构（V1.0完整版）

### 3.1 用户表（users）

```sql
CREATE TABLE users (
  id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '用户ID',
  phone VARCHAR(20) UNIQUE NOT NULL COMMENT '手机号',
  password_hash VARCHAR(255) NOT NULL COMMENT '密码哈希（bcrypt）',
  guardian_type ENUM('mechanic', 'elf', 'astrologer') NOT NULL COMMENT '守护灵类型',
  level INT DEFAULT 1 COMMENT '领地等级（1~5）',
  exp INT DEFAULT 0 COMMENT '当前经验值',
  invite_code VARCHAR(10) UNIQUE NOT NULL COMMENT '个人邀请码（6位字母数字）',
  invited_by BIGINT COMMENT '邀请人ID（FK -> users.id）',
  territory_coord_x INT NOT NULL COMMENT '领地X坐标（-1000~+1000）',
  territory_coord_y INT NOT NULL COMMENT '领地Y坐标（-1000~+1000）',
  signature VARCHAR(20) DEFAULT '' COMMENT '个性签名（20字）',
  gold_coins INT DEFAULT 100 COMMENT '金币余额（0~99999）',
  role ENUM('player', 'vip', 'club_admin', 'operator', 'super_admin') DEFAULT 'player' COMMENT '用户角色',
  status ENUM('active', 'frozen', 'deleted') DEFAULT 'active' COMMENT '账号状态',
  last_login_at DATETIME COMMENT '最后登录时间',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '注册时间',
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  UNIQUE KEY uk_coord (territory_coord_x, territory_coord_y),  -- 坐标不可重叠
  KEY idx_invite_code (invite_code),
  KEY idx_invited_by (invited_by),
  KEY idx_phone (phone)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

**关键约束**：
- `territory_coord_x`和`territory_coord_y`必须唯一（UNIQUE KEY uk_coord）
- `invited_by`为外键，引用`users.id`（需应用层保证，InnoDB支持外键）
- `invite_code`为6位字母数字，唯一

### 3.2 邀请记录表（invite_records）

```sql
CREATE TABLE invite_records (
  id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '记录ID',
  inviter_id BIGINT NOT NULL COMMENT '邀请人ID（FK -> users.id）',
  invitee_id BIGINT UNIQUE NOT NULL COMMENT '被邀请人ID（FK -> users.id）',
  invited_at DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '邀请时间',
  is_active BOOLEAN DEFAULT FALSE COMMENT '是否活跃（被邀请人活跃7天后变为true）',
  mini_program_generated BOOLEAN DEFAULT FALSE COMMENT '是否已生成小程序',
  KEY idx_inviter_id (inviter_id),
  KEY idx_invitee_id (invitee_id),
  FOREIGN KEY (inviter_id) REFERENCES users(id) ON DELETE CASCADE,
  FOREIGN KEY (invitee_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='邀请记录表';
```

**关键约束**：
- `invitee_id`必须唯一（一个用户只能被一个邀请人邀请）
- `is_active`默认为false，被邀请人活跃7天后变为true（通过定时任务更新）
- `mini_program_generated`防止重复生成小程序

## 4. API接口清单（V1.0完整版）

> 基础URL：`https://api.gamden.com/api`  
> 认证方式：Bearer Token（JWT），需在Header中携带`Authorization: Bearer <token>`  
> 响应格式：`{ "code": 200, "message": "success", "data": {...} }`

### 4.1 认证模块

#### 4.1.1 用户注册
```
POST /api/auth/register
```

**请求参数**：
```json
{
  "phone": "13800138000",           // 手机号
  "password": "abc123",             // 密码（明文，后端bcrypt加密）
  "invite_code": "A3B7C9",         // 邀请码
  "guardian_type": "mechanic"       // 守护灵类型（mechanic/elf/astrologer）
}
```

**返回数据**：
```json
{
  "code": 200,
  "message": "注册成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",  // JWT Token
    "refresh_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user_id": 123456,
    "territory_coord_x": 100,
    "territory_coord_y": 200
  }
}
```

**错误码**：
- 400：参数错误（手机号格式错误、密码长度不足等）
- 1001：邀请码无效
- 1002：邀请码已使用
- 1005：领地坐标已被占用（极少发生，因系统自动分配）

#### 4.1.2 用户登录
```
POST /api/auth/login
```

**请求参数**：
```json
{
  "phone": "13800138000",
  "password": "abc123"
}
```

**返回数据**：同注册接口

#### 4.1.3 刷新Token
```
POST /api/auth/refresh
```

**请求参数**：
```json
{
  "refresh_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

**返回数据**：
```json
{
  "code": 200,
  "message": "刷新成功",
```

## 5. 第三方服务集成

### 5.1 腾讯云内容安全API

**用途**：UGC内容审核（用户签名、俱乐部帖子、回帖）

**接入方案**：
1. 注册腾讯云账号，开通内容安全服务
2. 获取SecretId和SecretKey
3. 在`.env`文件中配置：
   ```
   TENCENT_CLOUD_SECRET_ID=your_secret_id
   TENCENT_CLOUD_SECRET_KEY=your_secret_key
   ```
4. 调用腾讯云内容安全API（SDK或HTTP请求）
5. 解析返回结果（置信度、命中敏感词）

**API调用示例**（Node.js）：
```javascript
const tencentcloud = require('tencentcloud-sdk-nodejs');

const client = new tencentcloud.tms.v20201229.Client({
  credential: {
    secretId: process.env.TENCENT_CLOUD_SECRET_ID,
    secretKey: process.env.TENCENT_CLOUD_SECRET_KEY,
  },
  region: "ap-guangzhou",
});

async function checkContent(content) {
  const params = {
    Content: Buffer.from(content).toString('base64'),  // 内容需Base64编码
  };
  const response = await client.TextModeration(params);
  return response;  // { Suggestion: 'Pass'/'Review'/'Block', Confidence: 90, ... }
}
```

**关键规则**：
- 置信度>90%：直接通过（Pass）
- 置信度70%~90%：进入人工复审池（Review）
- 置信度<70%或敏感：拒绝发布（Block）

### 5.2 微信小程序API

**用途**：生成个人小程序码、分享小程序

**接入方案**：
1. 注册微信小程序账号，获取AppID和AppSecret
2. 在`.env`文件中配置：
   ```
   WECHAT_APPID=your_appid
   WECHAT_APPSECRET=your_appsecret
   ```
3. 调用微信小程序API生成小程序码（`wxacode.getUnlimited`）
4. 将生成的小程序码上传到COS（对象存储），获取URL

**API调用示例**：
```javascript
const axios = require('axios');

async function generateMiniProgramCode(user_id, page_path) {
  // 1. 获取access_token
  const token_response = await axios.get('https://api.weixin.qq.com/cgi-bin/token', {
    params: {
      grant_type: 'client_credential',
      appid: process.env.WECHAT_APPID,
      secret: process.env.WECHAT_APPSECRET,
    }
  });
  const access_token = token_response.data.access_token;

  // 2. 生成小程序码
  const code_response = await axios.post(
    `https://api.weixin.qq.com/wxa/getwxacodeunlimit?access_token=${access_token}`,
    {
      page: page_path,
      scene: `user_id=${user_id}`,
    },
    { responseType: 'arraybuffer' }  // 返回图片二进制数据
  );

  // 3. 上传到COS，获取URL
  const qr_code_url = await uploadToCOS(code_response.data);
  return qr_code_url;
}
```

### 7.2 性能监控（Prometheus）

**接入方案**：
1. 在后端代码中暴露`/metrics`端点，输出Prometheus格式的指标
2. 部署Prometheus Server，配置抓取任务
3. 部署Grafana，配置Prometheus数据源，创建仪表盘

**示例代码（Node.js）**：
```javascript
const prometheus = require('prom-client');

const httpRequestDuration = new prometheus.Histogram({
  name: 'http_request_duration_ms',
  help: 'Duration of HTTP requests in ms',
  labelNames: ['method', 'route', 'status_code'],
});

app.use((req, res, next) => {
  const start = Date.now();
  res.on('finish', () => {
    const duration = Date.now() - start;
    httpRequestDuration.observe(
      { method: req.method, route: req.route?.path, status_code: res.statusCode },
      duration
    );
  });
  next();
});

app.get('/metrics', (req, res) => {
  res.set('Content-Type', prometheus.register.contentType);
  res.end(prometheus.register.metrics());
});
```
