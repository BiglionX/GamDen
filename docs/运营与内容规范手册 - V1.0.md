### 1.3 语气规范

**机械师**：
- 语气：冷静、理性、系统化
- 用词：检测、信号、系统、校准、执行
- 避免：情绪化表达、感叹号过多

**精灵**：
- 语气：温柔、自然、亲切
- 用词：森林、树梢、清泉、新芽、~
- 避免：机械术语、过于正式的表达

**占星师**：
- 语气：神秘、深邃、宿命感
- 用词：星辰、星盘、命运、星力、轨迹
- 避免：现代术语、过于直白的表达

### 1.4 技术实现要点

- 话术模板直接写死在客户端或后台配置表（agent_templates表）
- 服务端根据触发事件和守护灵类型，从配置表读取对应话术
- 通过OpenIM自定义消息发送（带守护灵头像和语气）
- 所有Agent话术必须记录到agent_dialogues表，用于V2.0大模型训练数据分析

## 2. 内容安全边界

### 2.1 禁止内容清单

| 分类 | 说明 | 示例 | 处理方式 |
|------|------|------|----------|
| **政治敏感** | 政治人物、政治事件、敏感话题 | - | AI初审拒绝 + 人工复审 |
| **色情** | 色情词汇、色情描述 | - | AI初审拒绝 |
| **暴力** | 暴力、恐怖词汇、血腥描述 | - | AI初审拒绝 |
| **人身攻击** | 辱骂、歧视、人身威胁 | "你妈死了"、"智商低" | AI初审拒绝 + 人工复审 |
| **商业广告** | 广告、推广、外链 | "加QQ群"、"下载XX App" | AI初审拒绝 |
| **垃圾信息** | 刷屏、重复内容、无意义内容 | "哈哈哈哈"、"111111" | AI初审标记 |

### 2.2 审核标准

**AI初审**：
- 调用腾讯云内容安全API
- 置信度>90%：直接通过
- 置信度70%~90%：进入人工复审池
- 置信度<70%或敏感：拒绝发布

**人工复审**：
- 运营人员在后台查看待审核内容
- 审核操作：通过/拒绝
- 拒绝时：填写拒绝原因（必填），通知用户

**审核时效**：
- AI初审：实时（<1秒）
- 人工复审：24小时内（工作日）

### 2.3 案例库（示例）

| 内容 | AI初审结果 | 人工复审结果 | 说明 |
|------|------------|--------------|------|
| "今天天气真好" | 通过 | - | 正常内容 |
| "你妈死了" | 拒绝 | - | 人身攻击 |
| "加QQ群123456" | 拒绝 | - | 商业广告 |
| "哈哈哈哈哈哈" | 标记 | 拒绝 | 垃圾信息 |
| "原神启动！" | 通过 | - | 正常内容（游戏相关） |

### 2.4 敏感词库管理

**敏感词分类**（同后台管理系统需求文档4.3）：
- 政治敏感、色情、暴力、人身攻击、商业广告

**敏感词级别**：
- 高级别：直接拒绝（如政治敏感词）
- 中级别：进入人工复审（如人身攻击词）
- 低级别：标记警告（如垃圾信息）

**敏感词库更新**：
- 运营人员可添加、删除敏感词
- 每周审查敏感词库，根据新案例更新

## 3. 违规处理梯度

### 3.1 梯度规则

| 违规次数 | 处罚措施 | 执行方式 | 通知方式 |
|----------|----------|----------|----------|
| 第一次 | 警告 | 系统自动发送警告通知 | 站内信 + Push通知 |
| 第二次 | 禁言24小时 | 系统自动执行，用户无法发布内容 | 站内信 + Push通知 |
| 第三次 | 冻结账号 | 系统自动执行，需运营人工复核 | 站内信 + Push通知 + 短信 |

### 3.2 执行标准

**警告**：
- 内容：通知用户"您发布的内容包含敏感词，已被删除。请遵守社区规范。"
- 影响：仅通知，不影响账号使用

**禁言24小时**：
- 内容：通知用户"您因多次发布违规内容，已被禁言24小时。解禁时间：YYYY-MM-DD HH:MM:SS。"
- 影响：用户无法发布内容（签名、帖子、回帖），但可浏览

**冻结账号**：
- 内容：通知用户"您的账号因多次违规已被冻结。如有异议，请联系客服申诉。"
- 影响：用户无法登录

### 3.3 申诉流程

**用户申诉**：
1. 用户收到处罚通知后，点击"申诉"
2. 填写申诉理由（必填）
3. 提交申诉
4. 系统生成申诉工单（appeals表）

**运营复核**：
1. 运营人员在后台查看申诉工单
2. 复核操作：通过/拒绝
3. 复核通过：撤销处罚（警告/禁言/冻结）
4. 复核拒绝：维持处罚，通知用户

**复核时效**：
- 48小时内（工作日）

### 3.4 特殊情况处理

**恶意用户**：
- 表现：频繁发布违规内容、多次申诉
- 处理：超级管理员可永久冻结账号

**误判**：
- 表现：AI初审误判、人工复审误判
- 处理：运营复核通过后，撤销处罚，并向用户道歉

## 4. 社群运营SOP（拉新期）

## 5. 危机公关

### 5.1 负面舆情处理

**监测渠道**：
- 微信公众号评论区
- 微博（搜索"GamDen"）
- 贴吧/论坛（如NGA、知乎）
- 应用商店评论（如App Store、Google Play）

**处理流程**：
1. 发现负面舆情（如用户投诉、恶意差评）
2. 评估影响范围（小范围/大范围）
3. 小范围：运营人员直接回复，解决问题
4. 大范围：启动危机公关预案，发布官方声明

**回复话术模板**：
- "非常抱歉给您带来不好的体验！请您私信我们，我们会尽快解决您的问题。"
- "关于[问题描述]，我们已记录并反馈给技术团队，会在下个版本修复。感谢您的反馈！"

### 5.2 用户投诉处理

**投诉渠道**：
- 站内信（App内）
- 邮件（support@gamden.com）
- 微信公众号后台

**处理流程**：
1. 接收用户投诉
2. 分类投诉类型（Bug反馈/内容审核异议/账号问题/其他）
3. 分配给对应负责人（技术/运营/客服）
4. 处理投诉，解决问题
5. 回复用户，确认问题已解决

**处理时效**：
- Bug反馈：24小时内回复，72小时内修复
- 内容审核异议：48小时内复核
- 账号问题：24小时内回复

### 5.3 危机公关预案

**场景1：服务器宕机**
- 影响：用户无法登录、无法使用功能
- 预案：
  1. 立即通知技术团队排查问题
  2. 在公众号/核心用户群发布通知"服务器维护中，预计30分钟恢复"
  3. 修复后，发布补偿方案（如全服发放100金币）

**场景2：数据泄露**
- 影响：用户数据泄露，引发信任危机
- 预案：
  1. 立即修复漏洞
  2. 发布官方声明，说明情况、补救措施
  3. 补偿受影响用户（如全服发放500金币）
  4. 加强数据安全防护

**场景3：恶意差评/竞争对手攻击**
- 影响：应用商店评分下降，新用户流失
- 预案：
  1. 回复差评，解决问题（如"抱歉给您不好的体验，请联系我们解决"）
  2. 引导满意用户好评（如"如果您对GamDen满意，请在应用商店给我们好评~"）
  3. 如确认是竞争对手攻击，收集证据，法律途径解决
