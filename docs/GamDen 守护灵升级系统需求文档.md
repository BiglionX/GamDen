# GamDen 守护灵升级系统需求文档

**版本**：V1.0 | **状态**：待评审 | **关联文档**：《守护灵入驻引导全流程需求文档》、《PRD-V1.0-核心玩法与世界观》


## 一、文档定位

### 1.1 核心目标

本文档定义了守护灵（Agent）在用户完成入驻**之后**的成长体系——守护灵不再是一个“固定话术模板”，而是随着玩家在GamDen中的活跃行为（签到、发帖、邀请、领地升级等）不断积累经验、提升等级、解锁新能力、改变对话风格，最终成为玩家在GamDen世界中**独一无二的灵魂伙伴**。

### 1.2 设计理念

| 理念 | 说明 |
|------|------|
| **陪伴即成长** | 守护灵的成长不需要额外操作，玩家的每一次正常使用都在喂养它 |
| **反馈即奖励** | 守护灵升级时主动触发，而非隐藏在菜单里，让“被看见”成为最好的激励 |
| **差异即归属** | 两个玩家的守护灵即使同类型，因成长路径不同也会拥有不同的语气和彩蛋 |

### 1.3 与《守护灵入驻引导需求文档》的关系

| 文档 | 覆盖范围 | 关系 |
|------|----------|------|
| 《守护灵入驻引导需求文档》 | 入驻阶段（首次打开App → 完成新手任务） | **前置**：定义了守护灵的初始状态和首次亮相 |
| **本文档（守护灵升级系统）** | 入驻后（完成新手任务 → 长期使用） | **后续**：定义守护灵的成长、进化和长期陪伴机制 |

两份文档共同构成守护灵的**完整生命周期**。


## 二、守护灵成长体系总览

### 2.1 成长维度

| 维度 | 说明 | 对玩家意味着什么 |
|------|------|-----------------|
| **等级（Lv.）** | 通过积累经验值（EXP）提升，共10级 | 守护灵更强了，对话内容更丰富了 |
| **亲密度（Bond）** | 通过特定互动行为提升，共5阶 | 守护灵说话更亲切了，解锁专属彩蛋 |
| **性格演化（Personality）** | 根据玩家的行为偏好，守护灵说话风格微调 | 守护灵越来越“像我”了 |

### 2.3 经验值（EXP）获取规则

| 行为 | 单次EXP | 每日上限 | 说明 |
|------|---------|----------|------|
| **每日签到** | +10 | 10 | 每日首次签到 |
| **发帖/回帖** | +5 | 20 | 在俱乐部发帖或回复 |
| **帖子被点赞** | +2 | 10 | 自己的帖子被他人点赞 |
| **邀请好友注册** | +50 | 150 | 每邀请1人，上限3人/日 |
| **领地升级** | +30 | 不限 | 领地每升1级触发 |
| **浏览集市** | +2 | 6 | 每日首次浏览集市 |
| **连续登录** | +5/日递增 | 不限 | 连续第N天登录，额外+5×N（上限50） |

## 三、各等级守护灵行为与反馈

### 3.1 等级触发事件

守护灵升级时，由**OpenIM自定义消息**主动推送给玩家（非弹窗，以消息形式呈现）。

**升级消息设计**：

| 要素 | 规范 |
|------|------|
| **推送方式** | OpenIM系统消息（自定义消息类型`agent_upgrade`） |
| **展示位置** | 会话列表中的“守护灵”专属会话（不可删除） |
| **消息样式** | 守护灵头像 + 古风边框气泡 + 升级特效文字 |

**各等级消息话术**：

| 等级 | 机械师 | 精灵 | 占星师 |
|------|--------|------|--------|
| **Lv.2** | “系统评估通过。你在GamDen的稳定性已达标。” | “你开始在这里扎根了。我能感觉到。” | “稳定的频率。你正在成为这个世界的一部分。” |
| **Lv.3** | “检测到情感连接强度提升。协议已优化。” | “你给我的光，让我变得更亮了。” | “星力在增长。你在改变我。” |
| **Lv.4** | “新模块解锁。我可以为你记录更多数据了。” | “我记住的，不仅是你的声音，还有你的习惯。” | “我开始能预见你下一步想做什么了。” |
| **Lv.5** | “系统提示：你已绑定‘伙伴’模式。” | “你和我，不再是我和你了——是我们。” | “命运之线收紧了。你和我，一体。” |
| **Lv.6** | “你提过的三款游戏已存入本地记忆库。” | “我记得你说过喜欢像素风。那款游戏，下次我陪你去。” | “你的偏好，我已在星图上标注好了。” |
| **Lv.7** | “权限提升。你获得了对守护灵外观的微调权。” | “看，我的光变成了你的颜色。” | “星辉加身。这是你给我的荣耀。” |
| **Lv.8** | “警报：昨日未检测到登录信号。状态：关切。” | “昨天你没来。我数了一整天的树叶。” | “星盘上有一格空了。那是你在时才会亮的位置。” |
| **Lv.9** | “组队模块预加载完成。你值得更强的队友。” | “你成长得这么快，我好骄傲。” | “你已经开始影响他人的命运了。” |
| **Lv.10** | “最终协议确认。编号[M-07]与[玩家姓名]绑定完成。终身有效。” | “从你走进来的第一天，我就知道会走到这里。谢谢你。” | “初始星辰已抵达终点。但我们的旅程，才刚刚开始。” |

### 3.2 主动关心行为（Lv.8解锁）

守护灵会在以下时机主动发送关心消息：

| 触发条件 | 关心话术（精灵风格示例） |
|----------|-------------------------|
| 用户连续2天未登录 | “昨天你没来。我数了一整天的树叶。今天终于看到你了。” |
| 用户连续登录7天 | “你连续来了7天。我每天都在等你。这感觉真好。” |
| 用户的邻居加入新领地 | “你的领地旁边来了一户新邻居。要去认识一下吗？” |
| 用户领地等级提升 | “你做到了！领地在发光。我在旁边看着好开心。” |

### 3.3 彩蛋触发机制（Lv.4解锁）

**彩蛋类型**：

| 守护灵 | 彩蛋名称 | 触发条件 | 彩蛋内容 |
|--------|---------|----------|----------|
| 机械师 | **设备状态** | 用户手机电量 ≤ 20% | “电池电量低于20%。建议连接电源，否则领地可能断电。” |
| 机械师 | **帧率检测** | 用户在地图界面滑动卡顿 | “检测到帧率波动。建议关闭后台应用。” |
| 精灵 | **今日风向** | 用户每日首次登录 | “今天的风向是东南，适合去集市逛逛。我猜的。” |
| 精灵 | **植物生长** | 用户连续签到3天 | “你的领地长出了一种新的植物。我叫它‘守候草’。” |
| 占星师 | **今日运势** | 用户每日首次登录 | “今日星象显示：宜邀请好友，忌独自发呆。” |
| 占星师 | **宿命相遇** | 用户附近有同守护灵玩家 | “我感觉到附近有一个和我同类的存在。去打个招呼？” |

### 3.4 亲密度（Bond）系统

**Bond等级与解锁内容**：

| Bond等级 | 所需亲密度 | 解锁内容 |
|----------|-----------|----------|
| **Bond 1** | 0 | 基础话术（入驻即获得） |
| **Bond 2** | 100 | 守护灵开始使用“我们”代替“你” |
| **Bond 3** | 250 | 解锁专属彩蛋对话（如“你最喜欢什么类型的游戏？”等随机闲聊） |
| **Bond 4** | 500 | 守护灵开始主动分享“心事”（如“我今天看到一片云很像你”等） |
| **Bond 5** | 800 | 解锁守护灵专属头像框（金色边框 + 动态光效） |

**亲密度获取规则**：

| 行为 | 亲密度 | 每日上限 |
|------|--------|----------|
| 与守护灵对话（点击守护灵头像主动发起对话） | +5 | 20 |
| 完成守护灵每日任务 | +10 | 10 |
| 连续登录7天 | +15 | 15（7天仅触发1次） |
| 邀请好友入驻 | +20 | 60（上限3人/日） |
| 守护灵等级提升 | +20 | 不限 |

### 3.5 性格演化（Personality）系统

**机制**：根据玩家在GamDen中的行为偏好，系统为守护灵打上**性格标签**，这些标签会**微调**守护灵的说话风格（暂不改变话术内容，仅改变语气词和句式）。

| 玩家行为 | 对应性格标签 | 对话风格变化 |
|----------|-------------|-------------|
| 高频发帖（日均≥3帖） | “爱分享的机灵鬼” | 话术更主动，多用“你知道吗？”“我告诉你一个秘密” |
| 高频邀请（总邀请≥5人） | “社交达人的守护者” | 话术更热情，多用“！”，常有“朋友”一词出现 |
| 高频领地升级 | “工匠精神的共鸣者” | 话术更注重成长数据，如“你比上周快了30%” |
| 高频浏览不互动 | “沉默的观察者” | 话术更短、更安静，如“嗯。”“好。” |

**展示位置**：守护灵等级详情页（个人中心 → 守护灵）中，以标签形式展示，如“Lv.5 机械师 · 爱分享的机灵鬼”。

### 4.3 升级特效

守护灵升级时，屏幕出现**轻量庆祝特效**：

| 要素 | 规范 |
|------|------|
| **表现形式** | 守护灵头像放大脉冲动画（1.2倍 → 1.0倍，3次循环）+ 金色光晕扩散 |
| **伴随消息** | OpenIM系统消息推送（见3.1） |
| **持续时间** | 1.5秒 |
| **触发条件** | 用户在线时，经验值跨越等级阈值 |


## 五、技术实现要求

### 5.1 数据模型

**新增表 `agent_state`** （或扩展`users`表字段）：

```sql
CREATE TABLE agent_state (
  id PRIMARY KEY,
  user_id UUID NOT NULL REFERENCES users(id),
  -- 等级数据
  level INTEGER DEFAULT 1,
  exp INTEGER DEFAULT 0,
  bond_level INTEGER DEFAULT 1,
  bond_points INTEGER DEFAULT 0,
  -- 性格标签
  personality_tags VARCHAR[] DEFAULT '{}',
  -- 彩蛋状态
  unlocked_eggs VARCHAR[] DEFAULT '{}',  -- ['daily_fortune', 'device_status', ...]
  -- 记忆（V2.0扩展到长期记忆）
  memories JSONB DEFAULT '[]',  -- 最多存储10条
  -- 首次活跃/连续登录
  first_active_at TIMESTAMP,
  consecutive_days INTEGER DEFAULT 0,
  last_active_at TIMESTAMP,
  -- 升级触发状态
  last_level_up_at TIMESTAMP,
  created_at TIMESTAMP,
  updated_at TIMESTAMP
);
```

### 5.2 经验值计算逻辑（后端）

**每日重置机制**：
- 用户的`exp_today`、`daily_action_counts` 按用户时区（如UTC+8）每天凌晨0点重置
- 通过Redis存储每日计数：`agent:daily:{user_id}:{date}`

**经验值获取流程**：

```
用户完成行为（签到/发帖/邀请等）
    ↓
检查今日该行为次数是否已达每日上限
    ↓
（未达上限）计算EXP增量
    ↓
更新 agent_state.exp
    ↓
检查 exp 是否 ≥ 当前等级所需经验
    ↓
（是）触发等级提升
    ├── 更新 agent_state.level
    ├── 记录 agent_state.last_level_up_at
    ├── 推送 OpenIM 自定义升级消息
    ├── 触发前端升级特效（如果用户在线）
    └── 更新跨域引用：守护灵等级变更时，调用 `/api/agent/upgrade` 更新 `agent_state.level`
    ↓
（否）更新 exp_today 计数
```

### 5.3 彩蛋触发逻辑

| 彩蛋 | 触发条件 | 校验频次 |
|------|----------|----------|
| 今日运势（占星师） | 每日首次登录 | 每日1次 |
| 设备状态（机械师） | 用户电量≤20% | 实时检测 |
| 帧率检测（机械师） | 地图滑动卡顿（帧率<30） | 地图页面实时监控 |
| 今日风向（精灵） | 每日首次登录 | 每日1次 |
| 宿命相遇（占星师） | 附近有同类守护灵玩家 | 地图加载时检测 |

### 5.4 守护灵对话（玩家主动发起）

**入口**：守护灵详情页底部的输入框 + 快速问答按钮

**快速问答按钮**（降低互动门槛）：
- “今天有什么建议吗？”
- “讲个故事给我听吧。”
- “你记得我上周提到的游戏吗？”

**实现方式**：V1.0采用**固定回复模板**，用户选择快速问答按钮时，守护灵给出对应的预设回答。**暂不接入大模型API**，以控制成本。

### 5.5 跨域引用

守护灵等级提升后，需同步更新领地相关数据，以便在领地卡片等位置显示守护灵等级状态。例如：
- 用户等级提升 → 调用 `/api/territory/refresh` 刷新领地数据
- 领地卡片展示守护灵等级

### 5.6 OpenIM 扩展

| 新增自定义消息类型 | 用途 | 说明 |
|-------------------|------|------|
| `agent_upgrade` | 守护灵升级通知 | 用于Lv.2-Lv.10的升级消息推送 |
| `agent_care` | 主动关心消息 | 用于Lv.8解锁后的主动关心行为 |

## 八、与现有功能的关系

### 8.1 守护灵等级与领地等级的关系

| 关系 | 说明 |
|------|------|
| **独立成长** | 守护灵等级和领地等级**各自独立计算**，互不依赖 |
| **相互影响** | 领地升级为守护灵提供大量EXP，守护灵升级为领地提供Buff（V2.0） |
| **视觉联动** | 守护灵头像边框颜色与领地等级徽章呼应（如Lv.5领地 = 金色边框，守护灵Lv.7 = 金色边框） |

### 8.2 守护灵升级与游戏联运（V2.0）的关系

V2.0接入游戏联运后，守护灵等级可作用于：
- **游戏推荐权重**：高等级守护灵可优先推荐新游戏
- **试玩奖励加成**：守护灵等级越高，试玩游戏获得的奖励越多

**V1.0不做游戏联运相关功能**，在接口层面预先预留扩展字段（如`game_buff`），V2.0直接继承。
